## 핵심 요약

- **모델 유형**: 1.6조 파라미터 Mixture-of-Experts(MoE) 오픈소스 언어 모델
- **핵심 기능**: 코딩, 추론, 웹 브라우징 에이전트, 롱컨텍스트 처리
- **가격(Pro)**: 입력 $1.74/백만 토큰, 출력 $3.48/백만 토큰 (캐시 미스 기준)

- **한국어 지원**: 공식 발표 없음. 다국어 LLM 특성상 한국어 입출력은 가능하나 최적화 수준은 별도 확인 필요

## DeepSeek-V4란?

DeepSeek-V4는 중국 퀀트 투자사 High-Flyer Capital Management의 AI 자회사 DeepSeek가 2026년 4월 24일 공개한 대형 언어 모델이다.

2025년 1월 오픈소스 R1 모델로 "미국 독점 AI 시대는 끝났다"는 평가를 받은 뒤, 커뮤니티는 차기 작품을 기다려 왔다. V3 시리즈 업데이트가 여러 차례 이어졌지만, 진정한 "두 번째 DeepSeek 순간"은 이번 V4가 처음이라는 것이 업계의 공통된 반응이다.

V4의 핵심 구조는 **Mixture-of-Experts(MoE)**다. 전체 파라미터가 1.6조 개에 달하지만, 실제 추론 시에는 활성화되는 파라미터만 선택적으로 작동해 연산 효율이 높다. 이 구조 덕분에 낮은 API 가격을 유지하면서도 높은 성능을 낼 수 있다.

경쟁 도구와 비교하면 차이가 선명하다.

| 모델 | 라이선스 | MoE 구조 | API 총비용(입력+출력) |
|---|---|---|---|
| DeepSeek-V4-Pro | MIT(오픈소스) | ✅ | $5.22 |
| GPT-5.5 | 비공개 독점 | 미공개 | $35.00 |
| [Claude](https://claude.ai) Opus 4.7 | 비공개 독점 | 미공개 | $30.00 |
| [Gemini](https://gemini.google.com) 3 Pro | 비공개 독점 | 미공개 | $14.00 |

오픈소스이면서 상업 라이선스(MIT)를 채택한 점도 중요하다. 기업이 자체 서버에 직접 배포할 수 있어 API 종속 없이 운영 비용을 더 낮출 수 있다.

DeepSeek AI 연구원 Deli Chen은 출시 당일 X(트위터)에 "V3 출시 484일 만에 완성한 노력의 결정체"이자 "AGI는 모든 사람의 것"이라고 밝혔다.


## 핵심 기능 5가지

### 1.6조 파라미터 MoE 아키텍처 — 비용과 성능의 동시 최적화

전통적인 밀집(dense) 모델은 추론마다 전체 파라미터를 활성화한다. MoE는 다르다. 수천억 개의 파라미터 중 각 토큰 처리에 필요한 전문가(expert) 서브네트워크만 선택해 실행한다.

결과적으로 GPT-5.5(비공개)처럼 단일 대형 모델과 비교할 때, 동등하거나 그 이상의 연산 효율을 낮은 비용으로 구현할 수 있다. 1.6조 파라미터라는 숫자가 크게 느껴지지만, 실제 추론에 동원되는 파라미터 수는 그보다 훨씬 적다. 바로 이 구조가 $1.74/백만 토큰이라는 가격을 가능하게 한다.

### BrowseComp 83.4% — 웹 에이전트 능력이 GPT-5.5 수준

웹 브라우징 에이전트 벤치마크인 BrowseComp에서 DeepSeek-V4-Pro-Max는 **83.4%**를 기록했다.

| 모델 | BrowseComp 점수 |
|---|---|
| GPT-5.5 Pro | 90.1% |
| GPT-5.5 | 84.4% |
| **DeepSeek-V4-Pro-Max** | **83.4%** |
| [Claude](https://claude.ai) Opus 4.7 | 79.3% |

단순 텍스트 생성이 아니라 실시간 웹 정보를 탐색하고 고도로 분산된 정보를 취합하는 능력을 측정하는 벤치마크다. [Claude](https://claude.ai) Opus 4.7보다 높고, GPT-5.5 기본 모델에 바짝 붙은 수치다. 특히 가격 대비 에이전트 성능으로는 현재 시장에서 가장 경쟁력 있는 선택지다.

### Terminal-Bench 2.0 67.9% — 코딩·터미널 작업도 준수

터미널 환경에서 AI가 명령어를 실행하고 문제를 해결하는 능력을 측정하는 Terminal-Bench 2.0에서 DeepSeek-V4-Pro-Max는 **67.9%**를 기록했다.

| 모델 | Terminal-Bench 2.0 |
|---|---|
| GPT-5.5 | 82.7% |
| [Claude](https://claude.ai) Opus 4.7 | 69.4% |
| **DeepSeek-V4-Pro-Max** | **67.9%** |

[Claude](https://claude.ai) Opus 4.7(69.4%)에 약 1.5%p 차이로 근접한 수준이다. GPT-5.5(82.7%)와는 격차가 있다. 코딩 자동화 파이프라인을 GPT-5.5 없이도 운영할 수 있는지를 판단하는 기준점으로 봐야 한다. 비용이 훨씬 낮다는 점을 고려하면, 중간 난이도의 터미널 작업에는 충분히 실용적이다.

### MIT 오픈소스 라이선스 — 자체 배포와 파인튜닝 가능

DeepSeek-V4는 MIT 라이선스로 배포된다. 이 라이선스는 상업적 이용, 수정, 배포를 모두 허용한다.

- **자체 서버 배포**: API 요금 없이 온프레미스 운영 가능
- **파인튜닝**: 자체 데이터로 모델을 추가 학습시킬 수 있음
- **Hugging Face 공개**: 누구나 즉시 다운로드 가능
- **재배포**: 수정 버전도 배포 허용

GPT-5.5나 [Claude](https://claude.ai) Opus 4.7은 API를 통해서만 접근 가능하다. 기업이 모델을 직접 통제하고 싶다면 DeepSeek-V4가 현실적으로 선택 가능한 거의 유일한 최상위급 옵션이다.

### V4-Flash — 거의 무료에 가까운 경량 옵션

DeepSeek-V4-Pro 외에 **DeepSeek-V4-Flash** 모델도 함께 출시됐다.

| 항목 | V4-Flash | V4-Pro |
|---|---|---|
| 입력 (캐시 미스) | $0.14/백만 토큰 | $1.74/백만 토큰 |
| 출력 | $0.28/백만 토큰 | $3.48/백만 토큰 |
| 합계 | $0.42 | $5.22 |
| GPT-5.5 대비 | 약 98% 저렴 | 약 85% 저렴 |

Flash 모델은 GPT-5.5 대비 약 98% 저렴하다. 다만 원문에서 명시됐듯이 "성능은 상당히 낮아진다(performance dips significantly)". 간단한 분류, 요약, 키워드 추출 같은 대량 배치 작업에는 Flash가, 복잡한 추론이 필요한 작업에는 Pro가 적합하다.


## 직접 써본 솔직한 후기

### 장점 3가지

**① 가격 대비 성능 비율이 현재 시장 최고 수준이다**

GPT-5.5와의 BrowseComp 점수 차이는 1%p(83.4% vs 84.4%)에 불과하다. 하지만 API 비용은 약 1/6 수준이다. 동일한 예산으로 GPT-5.5보다 6배 많은 토큰을 처리할 수 있다는 뜻이다. 대규모 추론 워크로드를 운영하는 기업이라면 이 차이가 연간 수억 원의 비용 절감으로 이어질 수 있다.

**② MIT 라이선스로 진짜 소유권이 생긴다**

OpenAI나 Anthropic의 API는 언제든 가격이 오를 수 있고, 서비스가 중단될 위험도 있다. DeepSeek-V4는 Hugging Face에서 직접 다운로드해 자체 인프라에 올릴 수 있다. 외부 API 의존도를 줄이고 싶은 기업에게는 결정적인 장점이다.

**③ 웹 에이전트 능력이 실제로 쓸 만한 수준이다**

BrowseComp 83.4%는 단순한 숫자가 아니다. 고도로 분산된 웹 정보를 탐색하고 취합하는 작업에서 [Claude](https://claude.ai) Opus 4.7(79.3%)을 앞서는 성능이다. 리서치 자동화, 경쟁사 모니터링 같은 에이전트 파이프라인에서 실용적인 선택지가 된다.

### 단점 / 아쉬운 점

**① GPQA Diamond와 HLE에서 GPT-5.5, Opus 4.7에 뒤진다**

복잡한 과학 추론 벤치마크인 GPQA Diamond에서 DeepSeek-V4-Pro-Max는 90.1%를 기록했다. GPT-5.5는 93.6%, [Claude](https://claude.ai) Opus 4.7은 94.2%다. Humanity's Last Exam(도구 없음)에서도 DeepSeek-V4는 37.7%로, GPT-5.5(41.4%)와 Opus 4.7(46.9%)에 뚜렷하게 뒤진다. 고난이도 과학·의학·법률 분야에서 최고 정확도가 필요한 작업에는 여전히 GPT-5.5나 Opus 4.7이 우선이다.

**② Terminal-Bench에서 GPT-5.5와 14.8%p 격차가 있다**

코딩 자동화 파이프라인 중 가장 복잡한 터미널 작업에서 DeepSeek-V4(67.9%)는 GPT-5.5(82.7%)보다 14.8%p 낮다. 이 차이는 단순한 수치 차이가 아니라, 복잡한 멀티스텝 터미널 명령어 시퀀스에서 실제로 실패율이 높아진다는 의미다. GPT-5.5 수준의 코딩 자동화가 필요하다면 DeepSeek-V4-Pro로는 대체가 어렵다.

**③ 한국어 최적화 수준이 불분명하다**

공식 발표에 한국어 지원에 관한 내용이 없다. MoE 구조의 다국어 LLM이므로 한국어 입출력 자체는 가능하다. 그러나 한국어 추론 정확도, 맞춤법, 자연스러운 문체 등의 최적화 수준은 별도 테스트가 필요하다. 한국어 전용 서비스를 구축하려는 경우라면 충분한 사전 검증이 요구된다.
