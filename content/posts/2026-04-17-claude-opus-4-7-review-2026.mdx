## [Claude](https://claude.ai) Opus 4.7이란?

[Claude](https://claude.ai) Opus 4.7은 Anthropic이 2026년 4월 16일에 공개한 현재 일반 공개(Generally Available) 기준 가장 상위 모델이다.

Anthropic은 AI 안전성 연구를 핵심 철학으로 삼는 미국의 AI 스타트업으로, OpenAI 출신 연구자들이 설립했다. [Claude](https://claude.ai) 시리즈는 Haiku(경량), Sonnet(중간), Opus(최상위) 세 등급으로 나뉘며, Opus 4.7은 그 최상위 라인의 최신 버전이다.

주목할 점은 Anthropic이 더 강력한 모델인 "Mythos"를 이미 보유하고 있다는 사실이다. 그러나 Mythos는 사이버보안 취약점 탐지 능력이 너무 강력해 외부에 전면 공개하기 위험하다는 판단 아래, 소수의 기업 파트너에게만 제한적으로 제공되고 있다. 즉, Opus 4.7은 "안전하게 공개할 수 있는 가장 강력한 모델"이라는 위치에 있다.

경쟁 구도를 보면, Opus 4.7은 OpenAI의 GPT-5.4(2026년 3월 초 출시)와 Google의 [Gemini](https://gemini.google.com) 3.1 Pro(2026년 2월 출시)를 주요 경쟁 상대로 삼는다. 직접 비교 가능한 벤치마크에서 Opus 4.7은 GPT-5.4를 7:4 비율로 앞서고 있다. 격차가 좁다는 점이 오히려 솔직한 부분이다. 모든 카테고리에서 이기는 게 아니라, 에이전틱 코딩·지식 업무·고해상도 시각 추론에서 특히 강하고, 에이전틱 검색·다국어 Q&A·터미널 코딩에서는 GPT-5.4에 밀린다.

## 핵심 기능 5가지

### 자기 검증 루프 — "리거(Rigor)" 설계

Opus 4.7의 가장 핵심적인 변화는 모델이 스스로 검증 단계를 만들어낸다는 점이다.

Anthropic은 이를 "rigor(엄밀함)"라고 부른다. 단순히 답변을 생성하는 게 아니라, 작업 완료 전에 자체적으로 검증 절차를 설계하고 실행한다. 내부 테스트에서 모델이 Rust 기반 텍스트-음성 변환 엔진을 처음부터 구축한 뒤, 자체 생성한 오디오를 별도 음성 인식기에 통과시켜 Python 레퍼런스와 대조 검증하는 과정을 자율적으로 수행했다고 알려졌다.

이 기능이 의미 있는 이유는 에이전틱 워크플로에서 흔히 발생하는 "환각 루프"를 줄이기 때문이다. 이전 세대 모델들은 틀린 결과를 생성해도 자신 있게 완료 보고를 했다. Opus 4.7은 작업 완료 전에 한 번 더 멈추고 확인한다.

### 고해상도 멀티모달 — 3.75메가픽셀까지 처리

Opus 4.7은 이미지 처리 해상도를 최장변 기준 2,576픽셀까지 지원한다. 이는 약 3.75메가픽셀에 해당하며, 이전 버전 대비 약 3배 향상된 수치다.

시각 정확도 벤치마크(XBOW)에서 이전 54.5%에서 98.5%로 수직 상승했다는 점은 눈에 띄는 결과다. 고해상도 UI를 탐색해야 하는 컴퓨터 사용 에이전트나, 기술 도면·재무 차트에서 데이터를 추출해야 하는 분석 업무에서 실질적인 차이가 생긴다.

| 항목 | 이전 버전 | Opus 4.7 |
|---|---|---|
| 최대 이미지 해상도 | 약 1.25메가픽셀(추정) | 3.75메가픽셀 (2,576px) |
| 시각 정확도 (XBOW) | 54.5% | 98.5% |
| arXiv 시각 추론 (도구 포함) | 84.7% | 91.0% |

### effort 파라미터 — "생각의 깊이"를 조절한다

새로 도입된 `effort` 파라미터는 모델이 문제에 얼마나 깊이 사고할지를 개발자가 직접 조정할 수 있게 해준다.

기존에는 `high`와 `max` 두 단계만 존재했다. Opus 4.7에서는 그 사이에 `xhigh(extra high)` 단계가 추가됐다. 내부 데이터에 따르면 `max` 설정은 코딩 태스크에서 약 75% 정확도를 보이고, `xhigh`는 성능과 토큰 소비 사이의 균형점을 제공한다.

- **high:** 빠른 응답, 낮은 토큰 소비
- **xhigh (신규):** 성능-비용 균형 최적 구간
- **max:** 최고 성능, 높은 토큰 소비

실무에서는 대부분의 에이전틱 태스크에 `xhigh`를 기본값으로 쓰고, 최종 검증 단계나 복잡한 코드 생성에만 `max`를 사용하는 방식이 권장된다.

### task budget — 토큰 비용을 미리 통제한다

[Claude](https://claude.ai) API에 "task budget" 기능이 퍼블릭 베타로 도입됐다. 개발자가 특정 태스크에 소비될 최대 토큰 수를 사전에 설정할 수 있는 기능이다.

Opus 4.7처럼 "생각을 많이 하는" 모델은 같은 작업에도 토큰을 훨씬 많이 소모한다. task budget은 이 비용을 예측 가능하게 만들어준다. 비용이 예측 불가능하다는 게 에이전틱 AI 도입의 최대 장벽 중 하나였는데, 이를 직접 건드린 기능이다.

아직 퍼블릭 베타 단계이므로 정식 출시 후 세부 동작 방식은 공식 문서 확인이 필요하다.

### 엄격한 명령어 준수 — 프롬프트 라이브러리 점검이 필요하다

Opus 4.7은 지시사항을 문자 그대로 따른다. 이전 모델들은 모호한 프롬프트를 맥락에 맞게 유연하게 해석했다면, Opus 4.7은 정확히 입력된 텍스트만 실행한다.

이는 에이전틱 자동화에서는 장점이다. 의도하지 않은 확장 해석으로 인한 오류가 줄어든다. 그러나 기존에 사용하던 프롬프트 라이브러리가 "모호한 지시"를 전제로 작성돼 있다면, 예상치 못한 결과가 나올 수 있다. Anthropic도 이 점을 공식 경고로 명시했다.

기존 [Claude](https://claude.ai) 모델 기반으로 구축된 자동화 파이프라인을 Opus 4.7로 마이그레이션할 때는 프롬프트 전수 점검이 사실상 필수다.


## 직접 써본 솔직한 후기

### 좋았던 점

**① 에이전틱 코딩에서 체감 차이가 난다**

SWE-bench Pro에서 64.3% 해결률은 이전 Opus 4.6의 53.4%에서 약 11%p 오른 수치다. 복잡한 다단계 코딩 태스크에서 스스로 중간 검증을 하고 수정하는 흐름이 실제로 작동한다. 특히 긴 컨텍스트가 필요한 코드 리팩토링에서 완성도가 올라갔다는 느낌이 확실하다.

**② 고해상도 이미지 분석이 실용적인 수준이 됐다**

이전 버전에서 고해상도 UI 스크린샷을 넣으면 세부 텍스트를 못 읽는 경우가 많았다. Opus 4.7에서는 2,576픽셀까지 처리하면서 실제 컴퓨터 사용 에이전트 용도로 쓸 수 있는 수준이 됐다. 시각 정확도 54.5%에서 98.5%라는 수치가 과장이 아님을 체감할 수 있다.

**③ GDPVal-AA 지식업무 벤치마크 1위는 납득 가능**

Elo 1753으로 GPT-5.4(1674)를 약 80점 앞서는 수치다. 복잡한 재무 분석, 법률 문서 요약, 학술 논문 정리 같은 지식 집약적 업무에서 전반적인 일관성이 높다.

### 아쉬운 점

**① 에이전틱 검색에서 GPT-5.4에 분명히 밀린다**

에이전틱 검색 벤치마크에서 Opus 4.7은 79.3%, GPT-5.4는 89.3%다. 10%p 차이는 작지 않다. 실시간 정보 검색이 포함된 에이전트 태스크에서는 여전히 GPT-5.4 쪽이 더 나은 결과를 보인다.

**② 한국어 지원이 아직 부족하다**

다국어 Q&A 벤치마크에서 Opus 4.7은 GPT-5.4와 [Gemini](https://gemini.google.com) 3.1 Pro에 뒤처진다고 원문에서 명시했다. 구체적인 수치는 공식 발표 확인이 필요하지만, 한국어로 복잡한 업무 지시를 내릴 때 영어 프롬프트보다 품질이 떨어질 수 있다. 한국어 사용자는 이 점을 염두에 둬야 한다.

**③ 프롬프트 마이그레이션 비용이 숨어있다**

명령어를 문자 그대로 따르는 특성 때문에, 기존 [Claude](https://claude.ai) 기반 파이프라인을 그대로 Opus 4.7에 올리면 예상치 못한 동작이 발생할 수 있다. 대규모 자동화를 운영 중인 팀이라면 마이그레이션 비용이 생각보다 클 수 있다.

**④ task budget이 아직 베타다**

비용 통제의 핵심 기능인 task budget이 아직 퍼블릭 베타라는 점은 프로덕션 도입을 고려하는 팀에게는 불안 요소다. 정식 출시 전까지는 토큰 소비 예측이 완전히 안정적이지 않을 수 있다.
