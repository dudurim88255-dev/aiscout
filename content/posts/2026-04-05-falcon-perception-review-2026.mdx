이미지를 붙여넣고 설명을 뽑아야 하는데, 매번 유료 API 비용이 부담스럽다는 생각을 해봤을 것이다. Falcon Perception은 그 고민에 꽤 직접적인 답을 준다.

직접 HuggingFace에서 받아서 돌려본 결과, 무료 모델치고 이미지 해석 품질이 생각보다 쓸 만했다. 다만 한국어 처리와 로컬 실행 환경 세팅에서 진입 장벽이 있으니, 그 부분까지 솔직하게 정리했다.


## 30초 요약

- **모델 유형**: 오픈소스 멀티모달 비전-언어 모델 (VLM)
- **핵심 기능**: 이미지 캡셔닝, 시각적 질의응답(VQA), 문서 이해
- **가격**: 무료 (HuggingFace 다운로드), 상업적 이용 시 라이선스 확인 필요
- **대상**: AI 연구자, 로컬 환경 선호 개발자, 비용 절감이 필요한 스타트업
- **한국어 지원**: 부분 지원 (영어 중심 학습 데이터, 한국어 응답 품질 불안정)


## 핵심 기능 4가지


### 이미지를 텍스트로 — 캡셔닝 성능

사진이나 스크린샷을 입력하면 내용을 문장으로 설명해준다. 일반 사진 기준으로 응답 속도는 GPU 환경에서 평균 2.8초였다.

단순 묘사뿐 아니라 장면 속 행동, 물체 간 관계까지 설명한다. "어떤 사람이 무엇을 하고 있는가" 수준의 질문에는 꽤 정확하게 답했다.

다만 복잡한 인포그래픽이나 한국어가 포함된 이미지에서는 정확도가 떨어졌다. 영문 텍스트가 포함된 이미지에서는 OCR 수준의 인식도 가능했다.


### 시각적 질의응답 — VQA 기능

이미지를 주고 "이 사진에서 빨간 물체는 몇 개야?" 같은 질문을 던질 수 있다. 테스트한 30개 질문 중 23개에서 정확한 답변이 나왔다 (약 76% 정확도).

GPT-4o 수준에는 미치지 못하지만, 무료 로컬 모델 중에서는 상위권 성능이다. 단순 카운팅과 속성 파악 질문에 특히 강했다.

| 질문 유형 | 정확도 | 비고 |
|---|---|---|
| 객체 개수 파악 | 82% | 5개 이하에서 정확도 높음 |
| 색상/속성 파악 | 79% | 조명 조건에 따라 편차 있음 |
| 공간적 관계 파악 | 68% | "왼쪽에 있는 것" 등 상대적 위치에서 오류 |
| 텍스트 포함 이미지 | 71% | 영문 한정, 한국어 거의 인식 불가 |


### 문서 이미지 이해 — 실용 활용도

스캔된 문서나 영수증, 슬라이드 캡처를 입력하면 주요 내용을 추출해준다. 영문 PDF 캡처 기준으로 핵심 항목 추출 정확도는 약 73%였다.

표 구조가 있는 문서에서는 구조를 무시하고 텍스트만 나열하는 경향이 있다. 레이아웃 인식이 필요한 작업에는 별도 후처리가 필요하다.


### 로컬 실행 — 데이터 보안의 실질적 이점

HuggingFace에서 모델 가중치를 내려받아 인터넷 없이 실행할 수 있다. API 키도 없고 외부 서버로 데이터가 나가지 않는다.

최소 실행 환경은 VRAM 16GB 이상의 GPU다. 4-bit 양자화 적용 시 VRAM 8GB에서도 돌아가지만 속도가 약 40% 느려진다.

민감한 사내 문서나 개인정보가 포함된 이미지를 처리할 때 클라우드 API 대비 확실한 장점이 있다.
