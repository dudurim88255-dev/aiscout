"AI한테 이거 좀 대신 해줘"라고 말하고 싶었던 적, 한 번쯤 있었을 거다. 보고서 검색하고, 파일 옮기고, 폼 입력하는 그 반복 작업들. Holo3는 그 상상을 실제로 구현하려는 모델이다.

HuggingFace에서 공개된 이 모델, 직접 테스트해봤다. 결론부터 말하면 **가능성은 진짜**다. 단, 한국어 환경에서 쓰려면 아직 참을 게 많다. 어떤 기능이 되고 어떤 게 안 되는지 솔직하게 정리했다.


## 30초 요약

- **모델 유형**: Computer Use 특화 멀티모달 AI 에이전트
- **핵심 기능**: 브라우저 자동 조작, 화면 인식, 앱 내 클릭 및 입력 수행
- **가격**: 현재 HuggingFace를 통해 무료 접근 가능 (API 사용량에 따라 과금 구조 예상)
- **대상 사용자**: 반복 업무 자동화가 필요한 개발자, 리서처, 파워 유저
- **한국어 지원**: 부분 지원 — 한국어 명령 이해는 되나, 한국어 UI 환경 조작은 불안정


## 핵심 기능 5가지


### 1. 화면을 보고 직접 클릭한다

Holo3의 핵심은 스크린샷 기반 인식이다. AI가 현재 화면을 보고 어디를 클릭해야 할지 스스로 판단한다. 단순 매크로가 아니라, 버튼 위치가 바뀌어도 의미를 파악해 찾아낸다.

테스트에서 구글 검색 후 특정 링크를 열고 내용을 요약하는 작업을 시켰다. 중간에 팝업이 떠도 스스로 닫고 작업을 이어갔다. 이 부분은 인상적이었다.


### 2. 자연어 명령으로 멀티스텝 작업 수행

"이 폴더 안에 있는 PDF 파일 이름을 날짜순으로 정리해줘" 같은 명령을 이해하고 실행한다. 단발성 대화가 아니라 **여러 단계를 순서대로** 처리하는 에이전트 방식이다.

명령 하나로 평균 5~8단계의 행동을 수행했다. 중간에 오류가 생기면 스스로 재시도하는 복구 로직도 있다.


### 3. 브라우저 자동화 — 로그인 없이 가능한 수준까지

웹 폼 작성, 버튼 클릭, 스크롤, 텍스트 입력까지 브라우저 내 대부분의 조작을 수행한다. 로그인이 필요한 영역은 제한이 있지만, 공개 웹 환경에서는 꽤 안정적으로 동작했다.

| 브라우저 작업 | 성공 여부 |
|---|---|
| 구글 검색 및 링크 열기 | ✅ 안정적 |
| 웹 폼 텍스트 입력 | ✅ 안정적 |
| 드롭다운 선택 | ⚠️ 간헐적 오류 |
| 로그인 세션 유지 | ❌ 미지원 |
| 한국어 웹사이트 조작 | ⚠️ 불안정 |


### 4. 파일 시스템 접근 및 조작

로컬 파일을 열고, 이름을 바꾸고, 특정 폴더로 이동시키는 작업이 가능하다. 운영체제 수준의 조작이라 처음에는 낯설 수 있다. 샌드박스 환경에서 실행하는 것을 권장한다.

"다운로드 폴더에서 오늘 받은 파일만 골라 정리" 같은 작업을 실제로 수행했다. 처리 시간은 평균 12~18초였다.


### 5. 작업 로그 및 재실행 기능

Holo3는 각 행동을 단계별로 기록한다. 어떤 순서로 무엇을 클릭하고 입력했는지 로그로 남는다. 오류가 발생했을 때 어느 단계에서 멈췄는지 파악하기 쉽다.

같은 작업을 반복 실행할 때 이 로그를 재활용할 수 있다. 완전한 RPA(로봇 프로세스 자동화)까지는 아니지만, 방향성은 그쪽이다.
