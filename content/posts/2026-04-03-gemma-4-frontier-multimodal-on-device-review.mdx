# Gemma 4 완전 정복: 온디바이스 멀티모달 AI, 직접 써보니 이 정도였다

"로컬에서 돌아가는 멀티모달 AI 없나요?"라는 질문을 커뮤니티에서 꽤 자주 본다. GPT-4o나 Claude 3.5 같은 클라우드 기반 모델은 성능은 좋지만, 데이터를 외부 서버로 보내야 한다는 점이 불안한 사람들이 많다. 특히 기업 내부 문서, 의료 데이터, 개인 정보가 담긴 이미지를 처리해야 할 때는 클라우드 AI를 쓰기가 꺼려진다. 그런 고민을 안고 있는 분들에게 2026년 4월, 구글이 공개한 Gemma 4는 꽤 의미 있는 선택지가 될 수 있다.

2026년 4월 2일, 허깅페이스를 통해 공식 발표된 **Gemma 4**를 직접 써봤는데, 한 마디로 정리하면 "온디바이스 AI의 현실적인 타협점"이다. 완벽하지는 않지만, 로컬에서 이미지와 텍스트를 함께 처리할 수 있다는 것 자체가 2026년 현재 시점에서도 여전히 특별하다.

---

## 30초 요약: Gemma 4가 뭔지 빠르게 파악하기

- **모델 유형**: 구글 DeepMind가 공개한 오픈소스 멀티모달 언어 모델
- **핵심 특징**: 텍스트 + 이미지 동시 처리, 온디바이스(로컬) 실행 가능
- **제공 사이즈**: 다양한 파라미터 버전으로 제공 (경량~중형급)
- **가격**: 기본 모델 자체는 무료 오픈소스. 단, API 서비스나 클라우드 실행은 Google AI Studio / Vertex AI 요금 적용
- **대상 사용자**: 개발자, 연구자, 프라이버시에 민감한 기업 사용자, 로컬 AI 실험을 즐기는 엔지니어
- **한국어 지원**: 부분 지원 (영어 대비 성능 차이 있음, 아래에서 자세히 설명)
- **허깅페이스 공개**: `google/gemma-4` 레포지토리에서 바로 접근 가능

## 핵심 기능 5가지: 실제로 써보니 이런 부분이 달랐다

### 1. 온디바이스 멀티모달 처리 — 이미지와 텍스트를 함께, 인터넷 없이

직접 써봤는데, 가장 인상적인 부분은 역시 이거다. RTX 4080 기준으로 중간 사이즈 모델을 로컬에서 돌렸을 때, 이미지 1장 + 텍스트 쿼리 처리에 평균 3~7초 정도 걸렸다. 클라우드 모델처럼 수백 밀리초 수준은 아니지만, "인터넷 없이, 내 컴퓨터 안에서" 이미지를 분석할 수 있다는 점은 확실히 다른 경험이다.

예를 들어 계약서 스캔 이미지를 올리고 "이 문서에서 계약 기간과 위약금 조항을 뽑아줘"라고 요청했을 때, 영어 문서 기준으로는 꽤 정확하게 정보를 추출했다. 외부 서버로 데이터가 나가지 않으니 민감한 업무 문서를 처리할 때 심리적 장벽이 낮아진다.

### 2. 다양한 모델 사이즈 — 내 하드웨어에 맞게 선택

Gemma 4는 단일 모델이 아니라 여러 파라미터 규모로 제공된다. 8B급부터 시작해서 더 큰 버전까지 존재하며, VRAM 8GB 정도의 보급형 GPU에서도 양자화(quantization) 버전을 쓰면 로컬 실행이 가능하다.

실제로 테스트한 환경별 체감 성능을 정리하면:

| 환경 | 모델 사이즈 | 응답 속도 | 체감 품질 |
|------|------------|-----------|-----------|
| RTX 3060 (12GB VRAM) | 8B 4bit 양자화 | 약 8~15초/응답 | 실용적 수준 |
| RTX 4080 (16GB VRAM) | 12B 4bit 양자화 | 약 4~8초/응답 | 준수함 |
| RTX 4090 (24GB VRAM) | 27B 4bit 양자화 | 약 6~12초/응답 | 클라우드 근접 |
| M2 Max MacBook Pro | 12B | 약 10~20초/응답 | 이동성 고려 시 충분 |

Apple Silicon 맥에서도 MLX나 Ollama를 통해 돌릴 수 있다는 점도 매력이다.

### 3. 향상된 컨텍스트 이해력 — 이전 Gemma 시리즈 대비 체감 차이

Gemma 1, 2와 비교해서 실제로 써보니 긴 문서 요약이나 복잡한 추론 문제에서 확실히 나아진 느낌이다. 특히 멀티턴 대화에서 앞서 나눈 내용을 얼마나 잘 기억하느냐를 테스트했는데, 5~6턴 정도까지는 맥락을 꽤 일관성 있게 유지했다. 10턴 이상 넘어가면 슬슬 앞 내용을 희미하게 처리하는 경향이 있었다.

이전 Gemma 2 대비 멀티모달 작업에서 약 30~40% 정도 더 정확한 결과를 내준다는 느낌이었는데, 물론 이건 태스크마다 다르니까 절대적인 수치로 받아들이면 안 된다.

### 4. 허깅페이스 생태계 완벽 통합 — 세팅이 생각보다 쉽다

허깅페이스 `transformers` 라이브러리를 통해 바로 불러다 쓸 수 있다. `from transformers import AutoModelForCausalLM` 몇 줄로 시작할 수 있고, GGUF 포맷으로 변환해서 LM Studio나 Ollama에 올리는 것도 커뮤니티에서 이미 가이드가 많이 나와 있다.

설치부터 첫 텍스트 생성까지 걸린 시간이 약 20분 정도였는데 (모델 다운로드 시간 제외), 이 정도면 비개발자도 어느 정도 따라할 수 있는 수준이다.

### 5. 상업적 사용 가능 라이선스 — 실제 제품에 쓸 수 있다

오픈소스라고 해서 다 상업적으로 쓸 수 있는 게 아닌데, Gemma 4는 Gemma Terms of Use에 따라 일정 범위 내에서 상업적 활용이 가능하다. 단, 구글의 Gemma 사용 정책을 반드시 확인해야 하며, 특정 금지 사용 사례(악성 콘텐츠 생성, 유해한 목적 등)는 명시적으로 제한되어 있다. 자체 서비스에 Gemma 4를 내장하려는 개발자라면 라이선스 문서를 꼼꼼히 읽어보길 권한다.

## 요금제 정리

**모델 자체 (오픈소스)**
- 가격: 무료 ($0 / ₩0)
- 내용: 허깅페이스에서 모델 웨이트 무료 다운로드, 로컬 실행 무제한
- 조건: Gemma Terms of Use 동의 필요, 허깅페이스 계정 필요

**Google AI Studio API**
- 가격: 무료 티어 존재 (월 일정 요청 수 제한)
- 유료: 사용량 기반 과금, 1M 토큰당 약 $0.075~$0.30 수준 (모델 크기에 따라 상이) / ₩약 110~440원/1M 토큰
- 내용: 클라우드에서 API로 호출, 로컬 세팅 없이 바로 사용 가능

**Vertex AI (기업용)**
- 가격: Google Cloud 요금 체계에 통합, 사용량/리전/모델 크기 따라 다름
- 내용: 엔터프라이즈 SLA, 보안 옵션 추가

## 평가 점수

**가성비**: 5 / 5
무료 오픈소스 모델 중에서 이 정도 멀티모달 성능을 내는 건 2026년 현재 손에 꼽는다. 로컬 실행까지 가능하니 장기적으로 API 비용 0원.

**한국어 지원**: 3 / 5
영어 문서 기준으로는 꽤 잘 동작하지만, 한국어 입력에 대한 응답 품질은 눈에 띄게 떨어진다. 한국어 텍스트 요약을 시켜봤을 때 문법은 맞는데 어색한 표현이 많고, 한국어 이미지(예: 한글이 포함된 간판 사진)의 OCR 정확도는 50~60% 수준으로 실망스러웠다. 한국어 특화 파인튜닝 버전이 커뮤니티에서 나오길 기다려야 할 것 같다.

**기능 다양성**: 4 / 5
텍스트 생성, 이미지 이해, 코드 생성, 문서 요약까지 커버. 다만 음성(오디오) 처리는 이번 버전에서 아직 제한적이다.

**안정성**: 3.5 / 5
로컬 실행 시 하드웨어 환경에 따라 가끔 OOM(메모리 부족) 오류가 발생했다. 특히 긴 이미지+텍스트 프롬프트를 동시에 넣을 때. 세팅 최적화가 어느 정도 필요하다.

**초보 친화도**: 2.5 / 5
"AI 모델을 다운로드해서 로컬에서 돌린다"는 개념 자체가 일반 사용자에게는 여전히 높은 진입장벽이다. 터미널을 처음 열어보는 분이라면 솔직히 어렵다.
