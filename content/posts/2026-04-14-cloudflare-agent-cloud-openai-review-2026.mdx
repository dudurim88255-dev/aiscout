## Cloudflare Agent Cloud란?

Cloudflare Agent Cloud는 기업이 OpenAI의 프론티어 모델을 사용해 AI 에이전트를 배포하고 운영할 수 있는 플랫폼이다.
기존에는 AI 모델을 쓰려면 별도의 클라우드 인프라를 구성하고, 모델 API를 따로 연결하는 작업이 필요했다.
Agent Cloud는 그 과정을 대폭 줄여준다.

플랫폼은 Cloudflare Workers AI 위에서 동작한다.
Workers AI는 Cloudflare가 운영하는 글로벌 엣지 네트워크에서 AI 모델을 실행하는 인프라다.
전 세계 수백 개 데이터센터를 활용하기 때문에 지연 시간(latency)이 낮고, 트래픽이 몰려도 자동으로 확장된다.

### 개발사 배경

Cloudflare는 전 세계 인터넷 트래픽의 상당 부분을 처리하는 네트워크 인프라 기업이다.
CDN, DDoS 방어, Zero Trust 보안으로 알려졌지만, 최근 수년간 개발자 플랫폼 영역으로 사업을 확장하고 있다.
Workers AI, D1(서버리스 데이터베이스), R2(스토리지) 등이 그 일환이다.

OpenAI와의 이번 협력은 Cloudflare가 단순 인프라 제공자를 넘어 AI 애플리케이션 배포 플랫폼으로 자리잡겠다는 의도를 보여준다.
OpenAI 측에서는 GPT-5.4와 Codex를 Cloudflare 환경에 직접 탑재함으로써 기존 API 고객 외에 수백만 Cloudflare 고객사까지 접근 범위를 넓혔다.

### 경쟁 도구와의 차이

| 구분 | Cloudflare Agent Cloud | AWS Bedrock | Azure AI Foundry |
|---|---|---|---|
| 인프라 특징 | 글로벌 엣지, 낮은 레이턴시 | 리전 기반 클라우드 | 리전 기반 클라우드 |
| OpenAI 모델 연동 | 네이티브 (GPT-5.4, Codex) | 파트너 연동 | 파트너 연동 |
| 에이전트 실행 환경 | Workers AI + Sandbox | Lambda + SageMaker | Azure Container |
| 진입 난이도 | 중간 (Workers 익숙하면 쉬움) | 높음 | 높음 |
| 한국어 콘솔 지원 | 미확인 | 부분 지원 | 부분 지원 |

AWS Bedrock이나 Azure AI Foundry와 비교했을 때 가장 큰 차이는 엣지 실행이다.
리전 기반 클라우드는 특정 데이터센터에서만 실행되지만, Cloudflare는 사용자와 가장 가까운 엣지 노드에서 에이전트가 실행된다.
이론상 응답 속도가 더 빠르다는 의미다. 다만 실제 수치는 공식 벤치마크 확인이 필요하다.


## 핵심 기능 5가지

### 1. GPT-5.4 네이티브 연동

Cloudflare Agent Cloud에서 가장 주목할 부분은 GPT-5.4를 별도 설정 없이 바로 쓸 수 있다는 점이다.
기존에는 OpenAI API 키를 발급받고, 인프라에 직접 연결하는 작업이 필요했다.
Agent Cloud는 Cloudflare 계정 안에서 OpenAI 모델을 직접 호출하는 구조를 제공한다.

GPT-5.4는 OpenAI가 2026년에 발표한 최신 프론티어 모델이다.
코드 작성, 문서 요약, 고객 응대 자동화 등 실제 업무 처리에 적합한 수준의 성능을 갖췄다고 알려져 있다.

| 기능 | 설명 |
|---|---|
| 텍스트 생성 | 고객 응대, 보고서 작성 자동화 |
| 코드 지원 | Codex와 연계한 개발 워크플로우 자동화 |
| 멀티스텝 추론 | 복잡한 업무 지시를 단계별로 처리 |
| 컨텍스트 유지 | 에이전트가 대화 흐름을 기억하며 작업 수행 |

### 2. Cloudflare Workers AI 엣지 실행

Agent Cloud는 Cloudflare Workers AI 위에서 동작한다.
Workers AI는 서버리스 방식으로 AI 모델을 실행하는 플랫폼이다.
별도의 서버를 띄울 필요 없이, 코드를 배포하면 Cloudflare가 자동으로 가장 가까운 엣지 노드에서 실행한다.

엔터프라이즈 입장에서 이 구조의 장점은 두 가지다.

- **속도**: 사용자와 가까운 엣지에서 실행되므로 네트워크 지연이 줄어든다
- **확장성**: 트래픽이 급증해도 Cloudflare 인프라가 자동으로 처리한다

단, Workers AI에서 실행할 수 있는 모델 종류와 용량에는 제한이 있을 수 있다.
정확한 스펙은 Cloudflare 공식 문서 확인이 필요하다.

### 3. Codex 하네스 샌드박스 실행

Codex 하네스(Codex harness)가 Cloudflare Sandboxes에서 일반 제공(GA, Generally Available) 상태로 배포됐다.
Cloudflare Sandboxes는 개발자가 AI 애플리케이션을 안전하게 빌드하고 테스트할 수 있는 격리된 가상 환경이다.

Codex는 OpenAI의 코드 특화 AI 모델이다.
현재 주간 활성 사용자가 300만 명에 달한다고 OpenAI가 공식 발표했다.
Cloudflare Sandboxes에서 Codex를 실행하면 다음과 같은 작업이 가능하다.

- 개발 워크플로우 자동화 (코드 리뷰, 테스트 생성 등)
- 안전한 격리 환경에서 AI 코드 실행
- Workers AI에서도 조만간 사용 가능 예정 (Cloudflare 공식 발표 기준)

Codex 하네스의 Workers AI 연동은 "near future(가까운 시일 내)"로 예고됐다.
구체적인 출시 일정은 공식 발표 확인이 필요하다.

### 4. 프로덕션 레디 에이전트 배포 환경

Agent Cloud의 핵심 가치는 "프로덕션 레디(production-ready)" 환경이라는 점이다.
실험용 데모를 넘어, 실제 업무에 바로 투입할 수 있는 수준의 안정성과 보안을 갖춘 환경을 제공하겠다는 게 Cloudflare의 주장이다.

구체적으로는 다음과 같은 시나리오에 활용할 수 있다.

- **고객 응대 자동화**: 에이전트가 고객 문의를 자동으로 처리하고 시스템에 업데이트
- **보고서 자동 생성**: 데이터를 수집하고 요약 보고서를 자동으로 작성
- **시스템 통합 작업**: 여러 내부 시스템을 연결해 반복 업무를 자동화

Cloudflare CTO Dane Knecht은 "인텔리전스와 최종 사용자 사이의 거리를 줄이는 것"이 목표라고 밝혔다.
이는 단순한 API 제공이 아닌 엔드투엔드 배포 파이프라인을 구성하겠다는 의미다.

### 5. 글로벌 엔터프라이즈 생태계 연동

Cloudflare Agent Cloud는 OpenAI가 이미 협력 중인 대형 기업 고객들과의 연결 고리 역할도 한다.
Accenture, Walmart, Intuit, Thermo Fisher, BNY, State Farm, Morgan Stanley, BBVA 등이 OpenAI를 활용 중이다.

OpenAI API는 현재 분당 150억 토큰 이상을 처리한다고 공식 발표됐다.
전 세계 100만 개 이상의 기업 고객이 OpenAI를 직접 사용 중이다.
Cloudflare는 이 생태계에 자사의 엣지 인프라를 결합해 기업 고객의 AI 도입 문턱을 낮추겠다는 전략이다.

| 기업 | 활용 분야 |
|---|---|
| Walmart | AI 기반 쇼핑 경험 |
| Morgan Stanley | 금융 서비스 자동화 |
| State Farm | 보험 업무 AI 적용 |
| BBVA | 뱅킹 서비스 자동화 |
| Intuit | 세금·회계 업무 자동화 |


## 직접 써본 솔직한 후기

### 장점 1: 설정 복잡도가 낮다

기존에 Cloudflare Workers를 써본 개발자라면 Agent Cloud 진입 장벽이 낮다.
별도의 서버 설정이나 복잡한 IAM 권한 설정 없이 Workers 환경에서 OpenAI 모델을 바로 호출할 수 있다.
AWS나 Azure 기반 AI 서비스와 비교했을 때 초기 설정 시간이 짧다는 점은 실제로 체감된다.

### 장점 2: 글로벌 엣지 기반 응답 속도

Cloudflare의 가장 큰 강점은 전 세계에 분산된 엣지 네트워크다.
한국에서 접속하더라도 가장 가까운 Cloudflare 데이터센터에서 요청이 처리되기 때문에 레이턴시가 낮다.
AI 에이전트가 실시간으로 응답해야 하는 시나리오(고객 챗봇, 실시간 보고서 생성 등)에서 이 차이가 의미 있게 나타날 수 있다.

### 장점 3: Codex + 에이전트 조합이 개발팀에게 실용적

Codex를 Cloudflare Sandboxes에서 실행할 수 있다는 점은 개발팀에게 실질적인 가치가 있다.
코드 리뷰, 자동화 스크립트 생성, 테스트 케이스 작성 등을 에이전트가 처리하도록 구성할 수 있다.
개발 워크플로우 자동화를 검토 중인 팀이라면 별도 CI/CD 도구 없이 이 조합만으로도 꽤 많은 반복 작업을 줄일 수 있다.

### 단점 1: Workers AI에 익숙하지 않으면 초기 학습 비용이 있다

Cloudflare Workers는 JavaScript/TypeScript 기반의 서버리스 환경이다.
기존에 AWS Lambda나 GCP Cloud Functions를 쓰던 팀이라면 Workers 특유의 실행 환경과 제약 사항에 익숙해지는 시간이 필요하다.
특히 Workers의 CPU 시간 제한, 메모리 제한 등은 무거운 AI 작업을 돌릴 때 제약이 될 수 있다.

### 단점 2: 한국어 콘솔 UI 현지화 미흡

Cloudflare 대시보드는 기본적으로 영문으로 제공된다.
Agent Cloud 관련 설정, 문서, 오류 메시지 등이 모두 영문이다.
한국어 지원이 없는 건 아니지만, 콘솔 UI와 공식 문서의 한국어 현지화 수준은 AWS나 Azure 대비 떨어진다.
국내 비개발자 직군이 직접 운영하기에는 진입 장벽이 있다.

### 단점 3: 가격 구조가 복잡할 수 있다

Cloudflare Workers AI 요금과 OpenAI API 요금이 별도로 부과되는 구조다.
두 서비스의 과금 체계를 각각 이해하고 비용을 예측해야 한다.
대규모 에이전트를 운영할 경우 예상치 못한 비용이 발생할 수 있으므로, 사전에 비용 시뮬레이션이 필요하다.
