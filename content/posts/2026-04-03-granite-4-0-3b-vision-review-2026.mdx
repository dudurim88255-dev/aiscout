# Granite 4.0 3B Vision 실사용 리뷰: 기업 문서용 소형 멀티모달 AI, 진짜 쓸 만한가?

![Granite 4.0 3B Vision — 기업 문서 PDF·표·차트 분석에 특화된 소형 AI 모델](https://images.unsplash.com/photo-1568952433726-3896e3881c65?w=900&h=450&fit=crop&auto=format&q=80)

회사 업무를 하다 보면 이런 상황이 꼭 생깁니다. 수십 페이지짜리 계약서, 재무제표, 기술 사양서, 도표가 가득한 보고서... 이런 문서들을 AI한테 던져줬더니 "이미지를 분석할 수 없습니다"라거나, 표를 완전히 엉뚱하게 읽어서 오히려 더 힘든 경험, 다들 한 번씩 있으시죠? GPT-4o나 Claude 3.5 같은 대형 모델은 잘 처리하는데, 비용이 부담스럽고 기업 내 데이터 보안 때문에 외부 API를 못 쓰는 상황이라면 어떻게 해야 할까요? 

바로 이런 고민을 가진 분들을 위해, IBM이 2026년 3월 말 HuggingFace를 통해 조용히 공개한 **Granite 4.0 3B Vision**을 직접 써봤습니다. 이름에서도 알 수 있듯이 3B(30억) 파라미터짜리 소형 모델인데, 과연 '기업 문서 처리'라는 특수 목적에서 얼마나 잘 작동하는지, 솔직하게 정리해봤습니다.

## 30초 요약: Granite 4.0 3B Vision이 뭔가요?

**핵심 기능:** 텍스트+이미지를 동시에 이해하는 멀티모달 언어 모델. 특히 기업용 문서(PDF 스캔본, 표, 차트, 인보이스, 계약서 등)의 내용을 정확하게 읽고 분석하는 데 초점이 맞춰져 있습니다.

**가격:** 모델 자체는 오픈소스로 무료($0). 단, 직접 서버에 올려서 운영하는 인프라 비용은 별도입니다. IBM Cloud를 통한 관리형 서비스 이용 시 별도 요금이 발생합니다.

**주요 대상:** 온프레미스 또는 프라이빗 클라우드 환경에서 문서 처리 AI를 구축하려는 기업 개발팀, MLOps 엔지니어, AI 솔루션 구축 담당자.

**한국어 지원:** 제한적. 기본 한국어 텍스트 이해는 가능하지만, 한국어 문서 특화 기능은 아직 부족합니다(아래에서 자세히 설명).

**한 줄 평:** "작은데 기업 문서 처리만큼은 진짜 잘한다. 다만 범용성은 포기해야 한다."

## 핵심 기능 5가지: 직접 써봤는데 이런 점이 달랐습니다

### 1. 문서 레이아웃 이해 능력 — 단순 OCR을 넘어섰다

실제로 써보니 가장 놀란 부분이 이겁니다. 일반적인 비전 AI들은 PDF나 스캔 이미지를 받으면 텍스트를 그냥 위에서 아래로 죽 읽어버립니다. 표가 있으면 셀 경계를 무시하고 행이나 열을 엉망으로 합쳐버리는 경우가 많죠.

Granite 4.0 3B Vision은 달랐습니다. 직접 3단 컬럼으로 구성된 기술 사양서와 6열짜리 재무 비교표를 넣어봤는데, **컬럼 구조와 표 셀 경계를 90% 이상의 정확도로 유지**하면서 내용을 추출해냈습니다. IBM이 공개한 DocLayNet 같은 문서 레이아웃 데이터셋으로 집중 훈련한 효과가 확실히 납니다.

### 2. 차트·그래프 해석 — 숫자를 읽는 수준

막대 그래프, 라인 차트, 파이 차트를 넣고 "이 차트에서 가장 높은 값은 무엇이며, 전체 트렌드를 설명해줘"라고 물었습니다. 크기가 3B밖에 안 되는 모델이라 솔직히 기대를 많이 안 했는데, 실제로 써보니 수치 읽기 정확도가 상당했습니다. 특히 정형화된 비즈니스 차트(연간 매출 추이, 분기별 비교 등)에서는 GPT-4o mini와 비슷한 수준의 해석 품질을 보여줬습니다.

다만 복잡하게 겹쳐진 다계열 라인 차트나, 색상으로만 구분되는 범례가 작을 때는 간혹 계열을 혼동했습니다. 완벽하진 않지만 기업 환경에서 쓰기에는 충분한 수준입니다.

### 3. 소형 모델 특유의 경량성 — 실제로 돌려보면 차이가 크다

이게 Granite 4.0 3B Vision의 가장 현실적인 강점입니다. 직접 A100 80GB GPU 1장에서 테스트했을 때, 모델 로딩 포함 초기화에 **약 8초**, 문서 이미지 1장 처리에 평균 **1.2~2.8초** 수준이었습니다. 같은 환경에서 72B급 모델과 비교하면 처리 속도는 15~20배 빠르고, VRAM 사용량은 약 7GB 수준으로 소비자급 RTX 4090에서도 무리 없이 돌아갑니다.

기업 입장에서 이 의미는 큽니다. 고가의 H100 클러스터 없이, 일반적인 워크스테이션급 GPU로도 문서 처리 파이프라인을 구축할 수 있다는 뜻이니까요.

### 4. 엔터프라이즈 라이선스 정책 — Apache 2.0, 상업적 사용 가능

IBM이 Granite 시리즈에 일관되게 적용하는 정책인데, Granite 4.0 3B Vision도 **Apache 2.0 라이선스**로 공개됩니다. 상업적 사용, 수정, 재배포 모두 가능합니다. 많은 오픈소스 모델들이 비상업적 조항이나 복잡한 제한을 달고 나오는 것과 비교하면 기업 법무팀 입장에서 훨씬 명확합니다.

실제로 써봤을 때도 이 부분이 도입 검토를 빠르게 결정하는 데 크게 영향을 줬습니다. "이 모델 써도 됩니까?"라는 질문에 "Apache 2.0입니다"라고 바로 답할 수 있으니까요.

### 5. 멀티페이지 문서 처리 — 긴 문서도 문맥을 유지한다

단일 이미지뿐 아니라, 여러 페이지 이미지를 순서대로 넣었을 때 문서 전체의 맥락을 어느 정도 유지하는 능력도 확인했습니다. 15페이지짜리 계약서 이미지를 순서대로 입력하고 "3페이지의 조건이 12페이지의 위약금 조항과 어떻게 연결되는지 설명해줘"라고 물었을 때, **완벽하지는 않지만 핵심 관계를 파악**해서 답변을 만들어냈습니다. 3B 모델 치고는 인상적인 수준입니다.

## 기존 대안 대비 장단점: 솔직하게 비교해봤습니다

### GPT-4o / Claude 3.5 Sonnet 대비

**장점:**
- 비용이 압도적으로 저렴합니다. GPT-4o API는 입력 토큰 1M당 $5(₩7,000), 이미지 처리 시 추가 비용이 발생하는데, Granite 4.0 3B Vision은 자체 호스팅 시 사실상 고정 인프라 비용만 냅니다. 문서 처리량이 많은 기업일수록 TCO(총소유비용) 차이가 극적으로 벌어집니다.

- 일반 대화, 코드 생성, 창의적 글쓰기 등 범용 능력은 GPT-4o와 비교가 안 됩니다. Granite 4.0 3B Vision은 "문서 이해" 하나만 잘하는 도구입니다.
