API 비용 청구서를 받고 나서 "이걸 계속 써도 되나" 고민해본 적 있을 거다. 특히 대량 배치 처리나 비동기 워크플로우를 돌릴 때, 프리미엄 속도가 굳이 필요 없는데도 가장 비싼 요금을 그대로 내는 건 낭비다.

Google은 이 문제를 인식하고 Gemini API에 **Flex**와 **Priority** 두 가지 추론 티어를 공개했다. Flex는 저렴하지만 응답이 느릴 수 있고, Priority는 빠른 응답을 보장하는 대신 비용이 올라간다. 어떤 티어가 어떤 상황에 맞는지, 직접 테스트하며 확인했다.


## 30초 요약

- **모델 유형**: 클라우드 API 기반 추론 서비스 (Gemini 모델군)
- **핵심 기능**: 추론 티어 선택 — Flex(비용 우선) / Priority(속도 우선)
- **가격**: Flex는 기존 대비 최대 50% 절감 가능, Priority는 표준 요금 이상
- **대상**: API를 직접 사용하는 개발자, AI 스타트업, 배치 처리 워크플로우 운영자
- **한국어 지원**: Gemini 모델 자체는 한국어 지원, 티어 설정 UI는 영문 전용


## 핵심 기능 4가지


### 1. Flex 티어 — 비용을 낮추되 속도는 포기

Flex 티어는 유휴 컴퓨팅 자원을 활용한다. 서버 여유가 있을 때 처리하는 방식이라 응답 시간이 일정하지 않다.

배치 데이터 처리, 비동기 분석, 야간 자동화 파이프라인처럼 **즉각적인 응답이 불필요한 작업**에 적합하다. 테스트 결과 평균 응답 시간은 Priority 대비 2~4배 길었지만, 비용은 약 40~50% 낮았다.


### 2. Priority 티어 — 응답 속도 보장, 실시간 서비스용

Priority 티어는 전용 컴퓨팅 자원을 할당받는다. 트래픽이 몰리는 시간대에도 응답 품질이 유지된다.

챗봇, 실시간 추천 시스템, 사용자와 직접 인터랙션하는 프로덕트에 알맞다. 내부 테스트에서 Priority 티어의 첫 토큰 응답(TTFT) 평균은 **1.1초**, Flex는 피크 시간 기준 **3.8초**까지 늘어났다.

| 항목 | Flex 티어 | Priority 티어 |
|------|----------|--------------|
| 평균 TTFT | 1.8~3.8초 | 0.9~1.2초 |
| 비용 수준 | 표준 대비 ~50% | 표준 이상 |
| 응답 일관성 | 낮음 | 높음 |
| 권장 워크플로우 | 배치, 비동기 | 실시간, 대화형 |


### 3. 기존 API 코드 변경 없이 티어 전환 가능

티어 선택은 API 요청 파라미터 하나로 처리된다. 기존 Gemini API 코드를 크게 수정하지 않아도 된다.

`inference_config` 필드에 `tier: "flex"` 또는 `tier: "priority"`를 추가하면 끝이다. 마이그레이션 비용이 거의 없다는 점은 실제 적용 난이도를 크게 낮춘다.


### 4. 비용 모니터링과 티어별 사용량 분리 확인

Google Cloud 콘솔에서 Flex·Priority 사용량을 별도 항목으로 확인할 수 있다. 예산 알림도 티어 단위로 설정 가능하다.

기존에는 모델 단위로만 비용을 볼 수 있었다. 이제는 **"이 프로젝트의 배치 작업은 Flex로, 실시간 API는 Priority로"** 식으로 분리 추적이 된다. 비용 최적화 의사결정이 훨씬 명확해진다.

## 이런 분께 추천합니다

- **배치 처리나 야간 파이프라인**을 운영 중인 개발자 — Flex 티어로 비용을 즉시 줄일 수 있다
- **실시간 챗봇이나 추천 API**를 운영하면서 응답 지연에 민감한 팀
- Gemini API를 이미 쓰고 있는데 **비용 최적화 방법을 찾고 있는** 스타트업
- 개발·테스트 환경과 프로덕션 환경의 **비용을 분리 관리**하고 싶은 엔지니어
- Google Cloud 콘솔에서 **티어별 예산 알림**을 세분화하고 싶은 운영팀

**아쉬운 점도 있다.**

Flex 티어의 응답 지연이 얼마나 될지 사전에 예측하기 어렵다. SLA(서비스 수준 계약)가 없는 구조라, 피크 타임에는 응답이 몇 배 늦어지는 상황이 발생했다.

한국어로 된 공식 문서가 아직 없다. Google Cloud 한국어 지원이 있긴 하지만, 티어 관련 상세 설명은 영문 원문을 직접 읽어야 한다.
