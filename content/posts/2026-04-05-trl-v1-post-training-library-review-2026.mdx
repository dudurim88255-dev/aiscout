LLM을 직접 파인튜닝해보고 싶은데, 어디서 시작해야 할지 몰라서 며칠째 탭을 쌓아두고 있는 건 아닌가. 그 고민, 나도 해봤다. 결론부터 말하면, TRL v1.0은 그 탭들 중 대부분을 닫아도 될 만큼 실용적인 도구다.

이 글에서는 TRL v1.0이 어떤 기능을 제공하는지, 실제로 돌려봤을 때 어떤 느낌인지, 그리고 어떤 상황에서 써야 하는지를 직접 테스트한 기준으로 정리했다. 설치 방법부터 주요 트레이너 구성까지 순서대로 살펴본다.


## 30초 요약

- **모델 유형**: 오픈소스 Python 라이브러리 (LLM 사후 학습 전용)
- **핵심 기능**: SFT, DPO, PPO, GRPO, ORPO 등 최신 학습 알고리즘 통합 제공
- **가격**: 완전 무료 (오픈소스, Apache 2.0 라이선스)
- **대상**: LLM 연구자, ML 엔지니어, 파인튜닝에 관심 있는 개발자
- **한국어 지원**: 문서 및 UI 영어 전용. 한국어 모델 학습 자체는 가능


## 핵심 기능 6가지


### SFT Trainer — 지도 학습 파인튜닝의 시작점

SFTTrainer는 허깅페이스 Transformers와 완전히 호환된다. 데이터셋 포맷만 맞추면 몇 줄로 학습을 시작할 수 있다.

LoRA, QLoRA 같은 파라미터 효율 기법도 PEFT 라이브러리와 연결해서 바로 쓸 수 있다. 메모리가 부족한 환경에서도 파인튜닝이 현실적인 이유다.


### DPO Trainer — 선호 데이터 기반 정렬

DPO(Direct Preference Optimization)는 보상 모델 없이 인간 선호 데이터를 직접 학습에 반영하는 방식이다. TRL v1.0에서는 DPOTrainer가 안정화됐고, 배치 처리 속도도 이전 버전 대비 눈에 띄게 개선됐다.

선호/비선호 응답 쌍만 준비하면 된다. 데이터 구조가 단순해서 초반 진입 장벽이 낮다.


### GRPO Trainer — 강화학습 기반 추론 능력 향상

GRPO(Group Relative Policy Optimization)는 DeepSeek-R1 논문에서 주목받은 알고리즘이다. TRL v1.0에서 정식으로 통합됐고, 수학적 추론이나 코드 생성 능력을 올리는 데 효과적이다.

PPO보다 구현이 단순하고, 별도의 가치 네트워크 없이 동작한다. 추론 특화 모델을 만들 때 현실적인 선택지가 됐다.


### ORPO / SimPO — 더 가벼운 정렬 옵션

ORPO와 SimPO는 참조 모델 없이 정렬 학습을 하는 방식이다. GPU 메모리를 절약할 수 있어서 리소스가 제한된 환경에서 유리하다.

| 알고리즘 | 참조 모델 필요 | 특징 |
|----------|----------------|------|
| DPO | 필요 | 안정적, 범용 |
| ORPO | 불필요 | 메모리 효율 |
| SimPO | 불필요 | 더 단순한 구조 |
| GRPO | 불필요 | 추론 특화 |


### vLLM 통합 — 학습 중 추론 속도 병목 해결

TRL v1.0은 vLLM과의 통합을 정식 지원한다. 온라인 강화학습(예: GRPO 학습 루프)에서 추론 단계가 병목이 되는 문제를 실질적으로 줄여준다.

실제 테스트에서 vLLM 없이 돌렸을 때 대비 추론 처리 속도가 2~4배 빨라졌다. 대규모 학습 파이프라인에서 체감 차이가 크다.


### CLI 기반 학습 실행 — 코드 없이 바로 시작

v1.0부터 `trl sft`, `trl dpo` 같은 CLI 명령어로 학습을 실행할 수 있다. 설정은 YAML 파일로 관리하고, 스크립트 작성 없이 빠르게 실험할 수 있다.

```bash
trl sft --model_name_or_path meta-llama/Llama-3-8B \
        --dataset_name trl-lib/Capybara \
        --output_dir ./output
```

처음 파인튜닝을 시도하는 사람이라면 여기서 시작하는 게 가장 빠르다.

TRL은 완전 무료 오픈소스다. 유료 플랜이 없다. 다만 실제로 돌리려면 GPU 비용이 발생한다. 클라우드 GPU를 쓴다면 Lambda Labs나 RunPod 기준 A100 1장에 시간당 $1.5~$2.5(₩2,100~₩3,500) 수준이다.


## 이런 분께 추천합니다

- LLM을 특정 도메인에 맞게 파인튜닝하고 싶은 ML 엔지니어
- RLHF, DPO 등 정렬 기법을 연구하는 대학원생 및 연구자
- 허깅페이스 생태계를 이미 쓰고 있어서 연동이 자연스러운 팀
- GPU가 한 장뿐인 제한된 환경에서 QLoRA로 파인튜닝을 시도하는 개발자
- DeepSeek-R1 류의 추론 특화 모델을 직접 만들어보고 싶은 사람

아쉬운 점을 솔직히 짚으면 세 가지다.

첫째, 문서가 영어 전용이다. 한국어 번역도, 한국어 커뮤니티도 없다. 영어 기술 문서에 익숙하지 않으면 초반에 시간이 많이 든다.

둘째, 알고리즘 수가 늘어나면서 어떤 걸 써야 하는지 선택이 쉽지 않다. DPO인지 ORPO인지 GRPO인지, 논문을 어느 정도 읽지 않으면 판단하기 어렵다.

셋째, GPU 없이는 의미 없다. Colab 무료 플랜에서도 소형 모델은 돌아가지만, 실제 실험 수준의 작업은 A100 이상이 필요하다.
