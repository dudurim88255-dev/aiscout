계약서 스캔본, 세금계산서 이미지, 손으로 쓴 회의록. 이런 걸 텍스트로 옮기느라 시간 낭비한 경험, 분명히 있을 것이다.

IBM이 새로 공개한 Granite 4.0 3B Vision은 이 문제를 꽤 실용적인 수준에서 해결해준다. 오픈소스라 무료고, 3B 크기라 로컬에서도 돌아간다. 실제로 기업 문서 이미지 여러 장을 넣어봤는데, 기대 이상이었다.


## 30초 요약

- **모델 유형**: 멀티모달 LLM (텍스트 + 이미지 동시 처리)
- **핵심 기능**: 문서 이미지 OCR·분석, 표·차트 이해, 인보이스/계약서 파싱
- **파라미터 수**: 30억 (3B) — GPU 없이도 실행 가능한 수준
- **가격**: 완전 무료 (Apache 2.0 라이선스)
- **대상**: 기업 문서 자동화 팀, 백엔드 개발자, MLOps 엔지니어
- **한국어 지원**: 부분 지원 (영어 중심, 한국어 문서 정확도는 낮음)
- **출처**: HuggingFace 공개 모델 (IBM Granite 시리즈)


## 핵심 기능 5가지


### 문서 이미지를 구조화된 텍스트로 변환한다

스캔한 PDF, 사진으로 찍은 영수증, 화면 캡처본 등을 입력하면 텍스트를 추출해준다. 단순 OCR이 아니라 문서의 구조까지 파악한다. 표가 있으면 표 형태로, 항목이 나뉘어 있으면 항목별로 분리해서 출력한다.

테스트에서 영문 인보이스 이미지를 넣었을 때 품목명, 수량, 단가, 합계를 JSON 형태로 정확히 뽑아냈다. 응답 시간은 평균 4.1초였다.


### 표와 차트를 읽고 해석한다

이미지 속 데이터 표를 읽어서 수치를 설명하거나 비교해준다. 막대 그래프나 파이 차트도 어느 정도 인식한다. "이 표에서 가장 높은 값은?" 같은 질문에 맥락에 맞는 답을 준다.

다만 복잡한 다중 축 그래프에서는 오류가 발생했다. 단순한 2D 표와 기본 차트에서는 신뢰할 만한 수준이었다.


### 3B 크기로 로컬 실행이 가능하다

30억 파라미터 모델이라 VRAM 8GB 수준의 GPU에서도 돌아간다. CPU 전용 실행도 가능하지만 속도가 많이 느려진다. 클라우드 API 비용 없이 사내 서버에 배포할 수 있다는 게 핵심이다.

| 실행 환경 | 응답 속도 (이미지 1장 기준) |
|---|---|
| GPU (RTX 4080, VRAM 16GB) | 약 2.8초 |
| GPU (RTX 3060, VRAM 8GB) | 약 5.3초 |
| CPU only (M2 Mac) | 약 22초 |


### 기업 문서 특화 파인튜닝이 되어 있다

일반 멀티모달 모델과 달리, 계약서·이메일·보고서·재무제표 같은 기업 문서 형식에 맞게 학습되어 있다. IBM Granite 시리즈의 특징인 enterprise-ready 방향성이 여기서도 유지된다. 규정 준수(compliance), 개인정보 보호 문서 처리에 대한 설명도 함께 제공된다.

일반 이미지 이해보다는 문서 처리 정확도가 더 높게 나왔다. 자연 풍경 사진을 넣으면 평범한 설명이 나오지만, 기업 양식 이미지를 넣으면 항목을 정확히 잡아냈다.


### Apache 2.0 라이선스로 상업적 사용이 가능하다

HuggingFace에서 무료로 내려받을 수 있다. 상업적 사용, 수정, 재배포 모두 허용된다. 사용료, API 호출 비용, 사용량 제한이 없다.

스타트업이나 중소기업이 문서 자동화 파이프라인을 구축할 때 비용 부담 없이 시작할 수 있다는 점이 실질적인 강점이다.
