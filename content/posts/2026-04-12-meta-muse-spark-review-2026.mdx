오픈소스 Llama 모델에 익숙해져 있던 개발자라면, 이번 발표가 불편하게 느껴질 수 있다. Meta가 완전한 독점 모델을 내놓은 건 이번이 사실상 처음이기 때문이다.

결론부터 말하면, Muse Spark는 써볼 만하다. 단, "현재로서는" 이라는 단서가 붙는다. 지금 이 글에서는 공식 발표 내용, 벤치마크 수치, 실제 접근 방법까지 정리했다.


## 30초 요약

- **모델 유형**: 독점 멀티모달 추론 모델 (비오픈소스)
- **핵심 기능**: 시각적 체인 오브 소트, 멀티에이전트 오케스트레이션, 도구 사용 지원
- **가격**: 미공개 (현재 무료 접근 가능, API는 선택적 프리뷰 단계)
- **대상 사용자**: AI 앱 사용자, 개발자(API 프리뷰), 멀티모달 작업이 많은 사람
- **한국어 지원**: 공식 발표 없음 (정확한 지원 여부는 Meta 공식 발표 확인 필요)


## 핵심 기능 4가지


### 1. 시각적 체인 오브 소트 (Visual Chain of Thought)

기존 AI 모델들은 이미지와 텍스트를 따로 처리한 뒤 붙이는 방식이었다. Muse Spark는 처음부터 시각 정보를 내부 추론 과정에 통합해 설계됐다. 에스프레소 머신 구조를 분석하거나, 요가 자세를 동영상으로 교정하는 작업이 가능하다고 Meta는 설명한다.


### 2. Contemplating 모드 (멀티에이전트 병렬 추론)

여러 서브 에이전트가 동시에 추론하는 "Contemplating" 모드가 새로 도입됐다. 이 모드에서 Muse Spark는 Humanity's Last Exam 벤치마크에서 58%, FrontierScience Research에서 38%를 기록했다. Google [Gemini](https://gemini.google.com) Deep Think, OpenAI GPT-5.4 Pro와 경쟁하는 수준이라고 Meta는 주장한다.


### 3. 사고 압축 (Thought Compression)

강화학습 과정에서 모델이 "지나치게 오래 생각하면" 패널티를 받는 구조다. 덕분에 더 적은 추론 토큰으로 복잡한 문제를 해결한다. Meta에 따르면 Llama 4 Maverick 대비 컴퓨팅 자원을 10배 이상 적게 사용한다고 한다.


### 4. 도구 사용 및 멀티에이전트 오케스트레이션

단순한 텍스트 생성을 넘어, 외부 도구를 호출하고 여러 에이전트를 조율하는 기능이 포함됐다. Meta는 이를 "개인 초지능(Personal Superintelligence)"의 기반으로 설명한다. 실제 개발 환경에서의 활용 폭은 API 프리뷰 이후 더 구체화될 것으로 보인다.

**가성비**: 지금은 무료로 쓸 수 있어서 점수를 줬다. API 가격이 공개되면 다시 평가가 필요하다.

**한국어 지원**: 공식 발표에 한국어 언급이 없다. 실제 지원 여부는 직접 테스트하거나 Meta 공식 발표를 기다려야 한다.

**기능 다양성**: 시각 추론, 멀티에이전트, 도구 호출까지 갖췄다. 현시점 멀티모달 모델 중 가장 넓은 스펙이다.

**안정성**: 출시 초기라 장기 안정성은 아직 판단하기 이르다. Llama 4 때 벤치마크 조작 논란이 있었던 만큼, 독립 검증 결과를 더 지켜봐야 한다.

**초보 친화도**: Meta AI 앱으로 접근하면 어렵지 않다. API는 아직 선택적 접근이라 일반 사용자에게는 닫혀 있다.

**한 줄 결론**: Muse Spark는 벤치마크 수치만 보면 Meta의 확실한 컴백이다. 하지만 독점 모델로의 전환, 불투명한 가격, 제한된 API 접근은 Llama를 믿고 써온 커뮤니티에게 물음표를 남긴다. 지금 당장 써보고 싶다면 Meta AI 앱부터 시작하고, API가 열릴 때까지 판단을 유보하는 게 현실적이다.
