클라우드 AI를 쓸 때마다 "내 데이터가 서버로 올라가는 건 아닐까" 불안했던 적, 있을 거다. 구글이 2026년 4월 공개한 Gemma 4는 그 불안을 끊어준다. 인터넷 없이 내 기기에서 직접 돌아가는 온디바이스 멀티모달 AI다.

결론부터 말하면, 무료인데 이 정도면 충분히 쓸 만하다. 단, 하드웨어 요구 사항이 만만치 않고 한국어 지원은 아직 완벽하지 않다. 설치 방법, 실제 응답 속도, 한국어 성능까지 이 글에서 전부 다룬다.


## 30초 요약

- **모델 유형**: 오픈소스 멀티모달 LLM (텍스트 + 이미지 입력)
- **핵심 기능**: 온디바이스 실행, 이미지 이해, 긴 컨텍스트 처리 (최대 128K 토큰)
- **가격**: 완전 무료 (허깅페이스에서 다운로드)
- **대상**: 개발자, 프라이버시 중시 사용자, AI 연구자
- **한국어 지원**: 부분 지원 (영어 대비 품질 약 70% 수준)


## 핵심 기능 4가지


### 온디바이스 실행 — 내 컴퓨터가 곧 서버다

Gemma 4는 외부 API 없이 로컬에서 돌아간다. 데이터가 외부로 나가지 않으니 민감한 문서 작업에 적합하다.

실제로 인터넷을 끊고 테스트했을 때 응답이 정상 작동했다. 다만 최소 16GB RAM에 GPU가 있어야 쾌적하게 돌아간다.


### 멀티모달 입력 — 이미지도 읽는다

텍스트만이 아니라 이미지를 입력으로 받을 수 있다. 스크린샷을 붙여넣고 "이 오류 메시지 분석해줘"라고 하면 즉시 답한다.

| 입력 유형 | 지원 여부 | 비고 |
|-----------|-----------|------|
| 텍스트 | ✅ 완전 지원 | |
| 이미지 (JPG/PNG) | ✅ 지원 | 최대 해상도 제한 있음 |
| PDF | ⚠️ 부분 지원 | 이미지로 변환 필요 |
| 오디오 | ❌ 미지원 | |
| 영상 | ❌ 미지원 | |


### 128K 토큰 컨텍스트 — 긴 문서도 한 번에

한 번에 처리할 수 있는 텍스트 길이가 최대 128,000 토큰이다. 긴 계약서나 논문 전체를 넣고 요약을 요청할 수 있다.

GPT-4o의 기본 컨텍스트(128K)와 동급이다. 단, 컨텍스트가 길어질수록 로컬 환경에서 응답 시간이 눈에 띄게 늘어난다.


### 다양한 모델 사이즈 — 내 하드웨어에 맞게 선택

Gemma 4는 단일 모델이 아니다. 파라미터 크기별로 여러 버전을 제공한다.

| 모델 버전 | 파라미터 | 권장 VRAM | 응답 속도 (테스트 기준) |
|-----------|----------|-----------|-------------------------|
| Gemma 4-1B | 10억 | 4GB | 평균 1.2초 |
| Gemma 4-4B | 40억 | 8GB | 평균 2.1초 |
| Gemma 4-12B | 120억 | 16GB | 평균 3.8초 |
| Gemma 4-27B | 270억 | 24GB | 평균 7.4초 |

직접 측정한 값이다. M2 Pro MacBook Pro 16GB 환경에서 4B 모델 기준 평균 응답 시간은 2.1초였다.

가성비는 5점 만점이다. 무료인데 이 정도 성능이 나온다는 게 핵심이다.

초보 친화도는 낮다. 터미널 명령어 한 줄도 어색한 분이라면 설치 단계에서 막힐 가능성이 높다. Ollama 같은 GUI 툴을 함께 쓰면 그나마 낫다.

한국어 지원은 아직 아쉽다. 영어 질문 대비 한국어 응답 품질이 체감상 70% 수준이다. 복잡한 한국어 문장을 이해하는 데 종종 오류가 생긴다.
