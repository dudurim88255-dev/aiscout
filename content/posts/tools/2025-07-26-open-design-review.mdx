Open Design은 Anthropic의 Claude Design을 대체하는 로컬 퍼스트 오픈소스 디자인 도구다. 별도의 에이전트를 탑재하지 않고, 이미 개발자 로컬에 존재하는 코딩 에이전트(Claude Code, Codex, Cursor Agent, Gemini CLI, OpenCode, Qwen, GitHub Copilot CLI)를 19개 조합 가능한 스킬과 71개 브랜드급 디자인 시스템으로 연결해 디자인 엔진으로 전환한다.

## 무엇을 해결하나

Claude Design은 LLM이 산문 대신 디자인 아티팩트를 직접 생성하는 가능성을 보여줬지만, 클로즈드 소스·유료·클라우드 전용·Anthropic 모델 종속이라는 제약이 뚜렷했다. 셀프호스트도, Vercel 배포도, 에이전트 교체도 불가능하다. Open Design은 같은 아티팩트 중심 워크플로를 재현하되 BYOK(Bring Your Own Key) 원칙을 모든 레이어에 적용해 잠금을 제거한다. `pnpm dev` 한 줄이면 로컬에서 돌고, Vercel에 바로 배포할 수 있다.

## 핵심 기능 3가지

**1. 19개 조합형 스킬 + 71개 디자인 시스템** — "매거진 스타일의 시드 라운드 피치덱을 만들어 줘" 같은 자연어 명령을 던지면, 인터랙티브 질문 폼이 먼저 뜨고 에이전트가 다섯 가지 시각 방향 중 하나를 선택한다. 결정적 팔레트 라이브러리와 레이아웃 템플릿이 프리플라이트로 적용되므로 AI 특유의 '맥빠지는 기본 디자인'을 구조적으로 차단한다.

**2. 샌드박스 프리뷰 + 다중 포맷 내보내기** — 생성된 아티팩트는 샌드박스된 iframe에서 즉시 렌더링되며, HTML·PDF·PPTX로 내보낼 수 있다. 디자인 결과물을 바로 프레젠테이션이나 웹에 활용할 수 있어 별도 변환 과정이 필요 없다.

**3. 에이전트 무관 아키텍처** — Claude Code, Codex, Cursor, Gemini CLI, Qwen, Copilot 등 어떤 코딩 에이전트든 연결 가능하다. 에이전트가 디스크 위 프로젝트 폴더를 읽고, 5차원 자체 비평을 거쳐 최종 아티팩트를 출력하는 구조이므로 특정 모델이나 서비스에 종속되지 않는다.

## 설치 및 사용

```bash
git clone https://github.com/nexu-io/open-design.git
cd open-design
pnpm install
pnpm dev
```

로컬 서버가 뜨면 브라우저에서 접속해 자연어로 디자인을 요청한다. Vercel 배포도 `vercel --prod` 한 줄이면 끝난다. BYOK 설정은 환경 변수로 API 키만 지정하면 된다.

## 누구에게 유용한가

- 프로토타입·피치덱·랜딩 페이지를 빠르게 뽑아야 하는 스타트업 개발자
- 디자인 시스템을 코드 기반으로 관리하면서 AI 자동화를 붙이고 싶은 팀
- Claude Design의 워크플로가 마음에 들었지만 클라우드 종속·비용이 부담인 개인 개발자
- 사내 보안 정책상 로컬 퍼스트가 필수인 엔터프라이즈 환경

## 한계

스킬과 디자인 시스템이 풍부하지만, 결국 최종 품질은 연결한 코딩 에이전트의 성능에 크게 좌우된다. 복잡한 인터랙션이나 애니메이션 수준의 디자인은 아직 커버하기 어렵고, 71개 디자인 시스템도 서구권 브랜드 미학 중심이라 한국어 타이포그래피나 국내 브랜드 톤에 맞추려면 커스텀 작업이 필요하다. 프로젝트가 아직 초기 단계인 만큼 API 인터페이스 변경 가능성도 염두에 둬야 한다.
