# llm-wiki — LLM 세션 로그를 검색 가능한 위키로 변환하는 도구

llm-wiki는 Claude Code, Codex CLI, GitHub Copilot, Cursor, Gemini CLI 등 주요 AI 코딩 도구의 세션 히스토리를 자동으로 수집해 검색·인터링크가 가능한 정적 지식 베이스(위키)로 변환하는 Python 기반 오픈소스 도구다. Andrej Karpathy가 제안한 "LLM Wiki" 패턴을 실제 동작하는 소프트웨어로 구현했다.

## 무엇을 해결하나

AI 코딩 도구를 활발히 사용하는 개발자라면 로컬 디스크에 수백 개의 세션 트랜스크립트가 쌓여 있다. 문제는 이 기록을 다시 열어보는 일이 거의 없다는 것이다. 과거에 어떤 프롬프트로 문제를 해결했는지, 어떤 모델이 더 나은 결과를 냈는지, 프로젝트별로 토큰을 얼마나 소비했는지 — 전부 기록에 있지만 활용되지 않는다. llm-wiki는 이 "죽은 데이터"를 시각화하고 구조화해서 재활용 가능한 지식으로 전환한다.

## 핵심 기능 3가지

**1. 멀티 소스 세션 통합**: Claude Code, Codex CLI, Copilot, Cursor, Gemini CLI 다섯 가지 도구의 세션 포맷을 모두 파싱한다. 도구를 여러 개 혼용하는 환경에서도 하나의 위키로 통합 관리할 수 있다.

**2. 풍부한 시각화**: 프로젝트별 활동 히트맵, 도구 사용 차트, 토큰 소비량, 모델별 정보 카드, 도구 간 비교(vs-comparison) 등 다양한 대시보드를 자동 생성한다. 필터링 가능한 세션 테이블로 과거 작업을 빠르게 탐색할 수 있다.

**3. AI 소비 가능한 내보내기**: `llms.txt`, `llms-full.txt`, JSON-LD 그래프, 페이지별 `.txt` + `.json` 사이드카 파일을 생성한다. 다른 AI 에이전트나 MCP 서버가 위키를 직접 쿼리할 수 있도록 설계된 구조다.

## 설치 및 사용

Python 3.9 이상이 필요하다. 명령어 두 줄이면 끝난다.

```bash
# 초기 설정 (의존성 설치)
./setup.sh

# 빌드 후 로컬 서버 실행
./build.sh && ./serve.sh
# http://127.0.0.1:8765 에서 확인
```

`examples/demo-sessions/` 디렉토리에 데모 데이터가 포함되어 있어 실제 세션 없이도 기능을 확인할 수 있다. 자신의 세션 데이터를 연결하면 동일한 형태의 위키가 개인 작업 내용으로 생성된다. GitHub Pages 배포도 `master` 푸시 시 자동으로 처리된다.

## 누구에게 유용한가

- AI 코딩 도구를 일상적으로 사용하면서 **세션 로그를 체계적으로 관리**하고 싶은 개발자
- 여러 LLM 도구를 병행 사용하며 **도구별 성능이나 토큰 효율을 비교**하려는 팀
- Karpathy의 LLM Wiki 패턴에 공감하며 **프롬프트 엔지니어링 노하우를 축적**하려는 실무자
- Obsidian과 연동해 기존 노트 시스템에 LLM 세션 컨텍스트를 통합하려는 사용자

## 한계

아직 v0.9.0 단계로 스타 수(58개)가 많지 않아 커뮤니티 검증이 충분하지 않다. 세션 파싱은 각 도구의 로컬 저장 포맷에 의존하므로, 도구 업데이트로 포맷이 변경되면 파싱이 깨질 수 있다. `raw/` 디렉토리가 gitignore 처리되어 있지만, 민감한 세션 데이터를 다루는 만큼 배포 전 개인정보 노출 여부를 직접 확인해야 한다. 런타임 의존성 추가를 금지하는 정책이 있어 확장성에는 제약이 따를 수 있다.
