Composio는 AI 에이전트가 외부 서비스(이메일, 캘린더, CRM, 개발 도구 등)와 상호작용할 수 있도록 타입 안전한 SDK를 제공하는 통합 플랫폼이다. TypeScript와 Python 양쪽 모두 지원하며, REST 클라이언트를 직접 작성할 필요 없이 프로덕션 수준의 서드파티 연동을 즉시 구현할 수 있다.

## 무엇을 해결하나

AI 에이전트를 만들 때 가장 번거로운 작업 중 하나가 외부 API 연동이다. OAuth 인증 흐름 처리, 각 SaaS별 API 스키마 파악, 에러 핸들링, 도구 정의 작성을 개별적으로 해야 한다. Composio는 이 과정을 하나의 SDK로 추상화한다. 특히 **Model Context Protocol(MCP)** 워크플로를 네이티브로 지원해, MCP 서버 구축 시 도구 등록과 인증 관리를 통합적으로 처리할 수 있다. LangChain, LlamaIndex, OpenAI Agents, Anthropic, Vercel AI SDK 등 주요 LLM 프레임워크와의 연동 레이어가 이미 준비되어 있어 프레임워크 전환 비용도 낮다.

## 핵심 기능 3가지

**1. 멀티 프레임워크 도구 바인딩** — 하나의 도구 정의로 OpenAI Agents, LangChain, LangGraph, LlamaIndex, Google Gemini, Vercel AI SDK 등 다양한 프레임워크에 동일한 액션을 주입할 수 있다. 프레임워크별 어댑터가 내장되어 있어 보일러플레이트가 거의 없다.

**2. OAuth·인증 관리 내장** — 서드파티 앱 연결 시 OAuth 2.0 플로우, API 키 관리, 연결된 계정 상태 추적을 SDK 레벨에서 처리한다. 개발자가 토큰 리프레시 로직을 직접 구현할 필요가 없다.

**3. MCP 서버 네이티브 지원** — Model Context Protocol 기반의 도구 서버를 바로 구성할 수 있다. 웹훅 트리거, 컴포저블 액션 체인까지 지원해 복잡한 자동화 파이프라인 구축이 가능하다.

## 설치 및 사용 예시

TypeScript 환경:

```bash
npm install @composio/core
```

```typescript
import { Composio } from '@composio/core';

const composio = new Composio({
  apiKey: process.env.COMPOSIO_API_KEY,
});

// 도구 목록 가져오기, 에이전트에 바인딩
const tools = await composio.getTools({ apps: ['gmail', 'slack'] });
```

Python 환경:

```bash
pip install composio
```

```python
from composio import Composio

client = Composio(api_key="YOUR_API_KEY")
tools = client.get_tools(apps=["gmail", "slack"])
```

두 언어 모두 동일한 API 구조를 따르므로, 팀 내 언어가 혼재되어 있어도 일관된 패턴으로 작업할 수 있다.

## 누구에게 유용한가

- **AI 에이전트 개발자**: 외부 SaaS 연동이 필요한 에이전트를 빠르게 프로토타이핑하고 싶은 경우
- **MCP 서버 구축자**: 도구 등록·인증·트리거를 하나의 SDK로 관리하고 싶은 경우
- **자동화 파이프라인 설계자**: 이메일 → 슬랙 → CRM 같은 크로스앱 워크플로를 에이전트 기반으로 구현할 때
- **멀티 프레임워크 환경**: LangChain에서 시작해 OpenAI Agents로 전환하는 등 프레임워크 의존성을 낮추고 싶은 팀
