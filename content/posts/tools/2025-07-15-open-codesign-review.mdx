Open CoDesign은 프롬프트 하나로 프로토타입, 슬라이드, PDF 같은 디자인 결과물을 로컬 환경에서 생성하는 오픈소스 AI 디자인 도구다. Claude Design, v0 by Vercel, Lovable, Bolt.new, Figma AI 등 클라우드 기반 유료 서비스의 대안으로, MIT 라이선스 아래 BYOK(Bring Your Own Key) 방식을 채택해 특정 공급자에 종속되지 않는다.

## 무엇을 해결하나

AI 네이티브 디자인 도구 시장은 빠르게 성장하고 있지만, 대부분 구독 과금·클라우드 전용·단일 모델 강제라는 세 가지 제약을 공유한다. 이미 Claude Code나 OpenAI Codex API 키를 보유한 개발자라면, 같은 키로 디자인 프로토타이핑까지 처리할 수 있어야 합리적이다. Open CoDesign은 이 간극을 정확히 겨냥한다. 데이터가 로컬에 머물고, 모델 선택권이 사용자에게 있으며, 데스크톱 앱이라 오프라인 워크플로에도 대응 가능하다.

## 핵심 기능 3가지

**1. 멀티모델 BYOK 지원** — Claude, GPT, Gemini, DeepSeek, Kimi, GLM, Ollama, 그리고 OpenRouter 같은 OpenAI 호환 엔드포인트까지 폭넓게 연결된다. 기존 Claude Code / Codex API 키를 원클릭으로 임포트하면 90초 안에 세팅이 끝난다. 모델을 자유롭게 전환하면서 같은 프롬프트에 대한 결과물 품질을 비교할 수 있다.

**2. 프롬프트 → 완성형 아티팩트** — 빈 프롬프트에서 출발해 에이전트가 계획·코드 작성·셀프 체크를 거쳐 hover 상태, 탭, 빈 상태까지 포함된 프로토타입을 자동 생성한다. 단순 목업이 아니라 인터랙션이 살아 있는 결과물이 나온다는 점이 차별점이다.

**3. 로컬 퍼스트 + Electron 데스크톱** — 모든 데이터와 프로젝트 파일이 로컬 디스크에 저장된다. Electron 기반이므로 macOS, Windows, Linux를 모두 커버하며, 서버 의존 없이 독립적으로 동작한다.

## 설치 및 사용

```bash
# 레포 클론
git clone https://github.com/OpenCoworkAI/open-codesign.git
cd open-codesign

# 의존성 설치
npm install

# 개발 모드 실행
npm run dev
```

릴리즈 페이지에서 플랫폼별 빌드 바이너리를 직접 다운로드할 수도 있다. 실행 후 설정 화면에서 사용 중인 API 키를 입력하면 즉시 프롬프트 입력이 가능하다.

## 누구에게 유용한가

- 이미 Claude Code / OpenAI API 키를 보유한 개발자가 추가 구독 없이 디자인 프로토타이핑을 돌리고 싶을 때
- 사내 데이터 유출 우려 때문에 클라우드 AI 디자인 도구를 쓸 수 없는 팀
- 여러 LLM 모델의 디자인 생성 품질을 동일 프롬프트로 비교 평가하려는 리서처
- Figma AI나 v0 같은 도구의 가격 정책에 부담을 느끼는 인디 개발자·1인 창업자

## 한계

스타 1,203개 수준의 초기 프로젝트로, 커뮤니티와 플러그인 생태계가 아직 얇다. Electron 앱 특유의 메모리 점유 문제는 감수해야 하며, Figma 수준의 정교한 벡터 편집이나 협업 기능은 현재 범위 밖이다. 모델 품질에 전적으로 의존하기 때문에, 로컬 Ollama 모델 사용 시 결과물 퀄리티가 클라우드 대비 뚜렷하게 떨어질 수 있다. 중국어 README가 먼저 작성된 점에서 볼 수 있듯 영문·한국어 문서화는 아직 보강이 필요하다.
