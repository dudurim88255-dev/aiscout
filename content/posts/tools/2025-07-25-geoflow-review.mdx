GEOFlow는 AI 모델을 활용해 GEO(Generative Engine Optimization) 및 SEO 콘텐츠를 대량으로 생산·심사·퍼블리싱하는 오픈소스 시스템이다. PHP 7.4 이상과 PostgreSQL을 런타임으로 사용하며, Docker Compose로 전체 스택을 한 번에 띄울 수 있다.

## 무엇을 해결하나

AI로 글을 생성하는 건 이제 누구나 한다. 문제는 그 이후다. 프롬프트 관리, 소재 정리, 생성 작업 스케줄링, 초안 검수, 최종 퍼블리싱까지 이어지는 과정이 전부 수작업이면 규모를 키울 수 없다. GEOFlow는 이 전체 체인을 하나의 파이프라인으로 엮어 콘텐츠 운영 자동화를 실현한다.

모델 설정 → 소재 관리 → 작업 생성·스케줄링 → AI 생성 → 초안·심사·게시 → 프론트엔드 SEO 출력. 이 여섯 단계가 하나의 관리 화면 안에서 돌아간다. 콘텐츠 사이트를 빠르게 세우거나, 사내 콘텐츠 운영 백오피스를 구축할 때 즉시 도입 가능한 구조다.

## 핵심 기능 3가지

### 1. OpenAI 호환 멀티 모델 연동과 배치 작업

OpenAI 스타일 API를 따르는 모든 서비스를 모델로 등록할 수 있다. 하나의 시스템에서 여러 AI 서비스를 동시에 운용하며, 작업(Task)을 생성하면 스케줄러가 잡 큐에 넣고 Worker가 실제 생성을 수행한다. 실패 시 자동 재시도까지 처리되므로 수백 건 단위 배치도 안정적으로 돌릴 수 있다.

### 2. 소재 통합 관리 체계

제목 라이브러리, 키워드 라이브러리, 이미지 라이브러리, 지식 베이스, 프롬프트 세트를 한곳에서 관리한다. 콘텐츠 생성에 필요한 원재료가 흩어져 있으면 운영 비용이 기하급수적으로 늘어나는데, GEOFlow는 이를 구조화된 저장소로 묶어 작업 생성 시 즉시 참조할 수 있게 했다.

### 3. 초안-심사-게시 3단 워크플로

생성된 글은 바로 공개되지 않고 초안(Draft) → 심사(Review) → 게시(Publish) 단계를 거친다. 자동 게시 모드로 전환하면 심사 없이 바로 공개할 수도 있다. 프론트엔드에서는 SEO 메타 정보, Open Graph 태그, 구조화 데이터(JSON-LD)까지 자동 출력되어 검색 엔진 최적화가 기본 탑재된 셈이다.

## 설치 및 사용

Docker Compose 방식이 가장 빠르다.

```bash
# 레포 클론
git clone https://github.com/yaojingang/GEOFlow.git
cd GEOFlow

# 환경 변수 설정
cp .env.example .env
vi .env  # DB, AI API 키 등 필수 값 입력

# 전체 스택 기동 (Web + PostgreSQL + Scheduler + Worker)
docker compose --profile scheduler up -d --build

# 프론트엔드 확인
# http://localhost:18080

# 관리자 백오피스
# http://localhost:18080/geo_admin/
```

로컬 PHP 서버로도 구동할 수 있다. PHP 7.4 이상에 `pdo_pgsql`, `curl` 확장이 필요하다.

```bash
export DB_DRIVER=pgsql
export DB_HOST=127.0.0.1
export DB_PORT=5432
export DB_NAME=geo_system
export DB_USER=geo_user
export DB_PASSWORD=geo_password

php -S localhost:8080 router.php
```

CLI도 제공된다. `bin/geoflow` 명령으로 작업 생성, 초안 업로드, 심사·게시, 상태 조회를 터미널에서 바로 처리할 수 있다. 배포 자동화 파이프라인에 끼워 넣기 좋은 구조다.

## 누구에게 유용한가

- **콘텐츠 SEO/GEO 운영자**: AI 생성 글을 대량으로 찍어내되, 품질 심사 과정을 놓치고 싶지 않은 경우.
- **니치 사이트 빌더**: 특정 도메인 키워드를 타겟으로 자동 콘텐츠 사이트를 빠르게 세우려는 경우.
- **사내 콘텐츠팀**: 소재·프롬프트·결과물을 팀 단위로 관리하면서 워크플로를 표준화하고 싶은 경우.
