# Litmus – AI 프롬프트를 유닛 테스트하고, 모델 비용까지 줄이는 CLI 도구

Litmus는 LLM 프롬프트에 대해 유닛 테스트 수준의 pass/fail 검증을 수행하고, 여러 모델 간 품질·비용을 비교해 최적의 모델을 찾아주는 오픈소스 Python CLI 도구다. REST API에 Postman이 있고 프론트엔드에 Cypress가 있듯, LLM 기반 애플리케이션에는 Litmus가 그 자리를 채운다.

## 무엇을 해결하나

LLM을 프로덕션에 투입한 팀이라면 공통적으로 겪는 문제가 있다. 프롬프트를 한 글자 수정했을 뿐인데 엣지 케이스 15%가 깨지고, 모델 선택은 "GPT-4o가 좋다더라"는 감에 의존하며, 테스트 데이터를 만들 시간은 늘 부족하다. Litmus는 이 세 가지를 정면으로 겨냥한다. 프롬프트 변경에 대한 회귀 테스트, 데이터 기반 모델 비교, 그리고 비용 시뮬레이션까지 하나의 CLI로 묶었다.

## 핵심 기능 3가지

**1. `litmus run` – 프롬프트 회귀 테스트**
프롬프트에 대한 assertion을 정의하고 여러 모델에 걸쳐 일괄 실행한다. 50개 이상의 시나리오를 한 번에 돌려 pass/fail 결과를 확인할 수 있다. JSON Schema 기반 구조화 출력 검증도 지원한다.

**2. `litmus generate` – AI 기반 테스트 데이터 자동 생성**
5개의 시드 예시만 주면 AI가 다양한 엣지 케이스 시나리오를 자동 생성한다. Promptfoo나 DeepEval에 없는 기능으로, 테스트 데이터 작성에 소모되는 시간을 대폭 줄여준다. `--seed` 플래그로 few-shot 기반 생성도 가능하다.

**3. `litmus cost` – 비용 절감 시뮬레이션**
일일 호출량을 입력하면 모델별 월간·연간 비용을 계산하고, 테스트를 통과하는 가장 저렴한 모델을 추천한다. Claude Haiku에서 Llama 3.1 8B로 전환 시 연간 $6,570 절감 같은 구체적 ROI를 보여준다.

## 설치 및 사용

```bash
# 설치
git clone https://github.com/litmus-ai/litmus.git
cd litmus
pip install -e .

# API 키 설정
cp .env.example .env
# .env에 OPENAI_API_KEY, ANTHROPIC_API_KEY, HF_TOKEN 중 하나 이상 입력

# 프로젝트 초기화 → 테스트 실행
litmus init
litmus run
```

```bash
# 비용 시뮬레이션 (일 50,000 호출 기준)
litmus cost --volume 50000
```

데이터베이스도 클라우드 계정도 필요 없다. 로컬에서 바로 동작한다.

## 누구에게 유용한가

- LLM 기반 기능을 프로덕션에 배포하면서 프롬프트 변경의 영향도를 정량적으로 추적하고 싶은 백엔드 개발자
- 여러 모델을 실제 태스크 기준으로 비교해 비용을 최적화해야 하는 AI 엔지니어 또는 팀 리드
- CI/CD 파이프라인에 프롬프트 테스트를 통합하려는 MLOps 담당자
