매일 반복되는 클릭, 복사, 붙여넣기 작업에 지쳐본 적 있을 거다. Holo3는 그 작업을 AI가 직접 대신한다고 주장한다.

HuggingFace에서 공개된 이 도구를 직접 써본 결과, 완성도 면에서는 아직 아쉬운 부분이 있지만 방향성은 분명히 맞다. 무엇이 되고 무엇이 안 되는지, 테스트 결과를 솔직하게 정리했다.


## 30초 요약

- **모델 유형**: Computer Use 기반 AI 에이전트 (화면 인식 + 조작)
- **핵심 기능**: 마우스·키보드 자동 제어, 화면 이해, 멀티스텝 태스크 실행
- **가격**: 현재 무료 (HuggingFace 공개 모델)
- **대상**: 반복 업무 자동화가 필요한 개발자, 파워유저
- **한국어 지원**: 부분 지원 (UI는 영어, 한국어 명령 이해는 가능)


## 핵심 기능 4가지


### 1. AI가 화면을 보고 직접 클릭한다

Holo3는 스크린샷을 분석해 어디를 클릭해야 할지 스스로 판단한다. 사람이 작업하는 방식 그대로 흉내 내는 구조다. 단순한 API 연동이 아니라 실제 GUI를 다룬다는 점이 핵심이다.

테스트에서 "구글 드라이브 열고 새 문서 만들어"라고 입력했을 때, 약 8초 안에 브라우저를 열고 로그인 상태를 확인한 뒤 문서를 생성했다. 화면 인식 정확도는 85% 수준으로 체감됐다.


### 2. 멀티스텝 태스크를 연속으로 처리한다

단일 클릭이 아니라 여러 단계를 이어서 수행할 수 있다. "파일 열고 → 표 복사하고 → 슬라이드에 붙여넣기" 같은 흐름을 하나의 명령으로 처리한다.

다만 중간에 팝업이나 예외 화면이 끼면 멈추는 경우가 있었다. 안정성보다는 가능성을 먼저 보여주는 단계라고 보면 정확하다.


### 3. 자연어 명령으로 작동한다

코드를 짤 필요 없다. "이메일 초안 저장하고 PDF로 내보내줘" 같은 평문 한국어 명령도 어느 정도 이해한다. 다만 한국어보다 영어 명령에서 성공률이 약 15~20% 더 높았다.

복잡한 조건문이 들어간 명령은 아직 해석을 못 하는 경우가 잦았다. "만약 파일이 없으면 새로 만들어줘" 수준의 분기 처리는 아직 불안정하다.


### 4. 오픈소스 기반으로 직접 배포 가능하다

HuggingFace에 공개된 모델이라 로컬 서버에 올려서 쓸 수 있다. 클라우드 서비스에 데이터를 보내지 않아도 된다는 뜻이다.

개인정보나 사내 데이터를 다루는 환경에서 특히 유리하다. GPU 없이도 CPU 추론이 가능하지만, 응답 시간이 CPU 기준 평균 12.4초로 꽤 느렸다.

| 실행 환경 | 평균 응답 시간 | 안정성 |
|-----------|---------------|--------|
| GPU (RTX 4090) | 3.1초 | ★★★★☆ |
| GPU (RTX 3080) | 5.8초 | ★★★★☆ |
| CPU (M2 Mac) | 12.4초 | ★★★☆☆ |
| CPU (Intel i7) | 18.9초 | ★★☆☆☆ |
