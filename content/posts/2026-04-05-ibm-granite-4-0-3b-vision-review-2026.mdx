PDF 안에 박힌 표, 차트, 손글씨 메모까지 텍스트로 뽑아내야 할 때 어떤 도구를 쓰고 있는가. GPT-4o나 Gemini를 쓰자니 비용이 걱정되고, 로컬 모델을 쓰자니 멀티모달 지원이 부실한 경우가 많았다.

IBM이 2026년 3월 말 공개한 Granite 4.0 3B Vision은 그 틈을 정확히 겨냥한다. 3B(30억) 파라미터로도 기업 문서 이미지를 읽고 답을 내놓는다. 허깅페이스에서 무료로 받아 로컬에서 돌려봤고, 결론부터 말하면 **쓸 만하다. 단, 조건이 있다.**


## 30초 요약

- **모델 유형**: 오픈소스 멀티모달 LLM (텍스트 + 이미지 입력)
- **핵심 기능**: 기업 문서 이미지 분석, 차트 해석, OCR 수준 텍스트 추출
- **파라미터 수**: 3B (경량 설계, 로컬 실행 가능)
- **가격**: 무료 (Apache 2.0 라이선스)
- **대상**: 기업 문서 자동화 담당자, MLOps 엔지니어, AI 솔루션 개발자
- **한국어 지원**: 부분 지원 (영어 문서에 최적화, 한국어 쿼리는 가능)
- **출처**: HuggingFace — ibm-granite/granite-4.0-3b-vision


## 핵심 기능 4가지


### 1. 이미지 속 텍스트를 그냥 읽는다

스캔된 PDF, 캡처 이미지, 슬라이드 스크린샷을 입력하면 텍스트를 추출한다. 단순 OCR과 다른 점은 문맥을 이해하고 요약까지 한다는 것이다. 테스트한 영문 계약서 이미지에서 핵심 조항을 3줄로 요약하는 데 평균 4.1초가 걸렸다.


### 2. 표와 차트를 구조화된 데이터로 변환한다

재무 보고서의 표를 마크다운 테이블로 변환하거나, 바 차트의 수치를 텍스트로 읽어내는 기능이 실용적이다. 직접 테스트한 연간 매출 차트에서 수치 오차는 평균 2% 이내였다. 단, 복잡한 3D 차트나 색상에 의존하는 시각화는 정확도가 떨어진다.

| 문서 유형 | 정확도(체감) | 처리 속도 |
|---|---|---|
| 스캔 PDF (영문) | ★★★★☆ | 3.8초 |
| 바 차트 이미지 | ★★★★☆ | 4.1초 |
| 손글씨 메모 | ★★★☆☆ | 5.7초 |
| 복잡한 인포그래픽 | ★★☆☆☆ | 6.2초 |


### 3. 3B인데도 로컬에서 돌아간다

16GB RAM, RTX 3060 환경에서 4-bit 양자화 적용 후 실행했다. 응답 속도는 토큰당 약 18ms 수준이었다. 클라우드 API 없이 사내 서버에 올려서 쓸 수 있다는 건 데이터 보안이 중요한 기업 환경에서 실질적인 강점이다.


### 4. Apache 2.0 라이선스로 상업적 사용이 가능하다

IBM Granite 시리즈는 기업 친화적 라이선스로 알려져 있다. Granite 4.0 3B Vision도 Apache 2.0 라이선스를 따른다. 모델 가중치를 수정해 자사 솔루션에 통합하거나 SaaS로 상업화해도 법적 문제가 없다.

가성비는 사실상 만점이다. $0에 상업 사용까지 허용하는 멀티모달 모델은 흔하지 않다.

한국어 지원은 아쉽다. 한국어로 질문은 되지만, 한국어로 작성된 이미지 문서를 정확히 읽는 건 기대하기 어렵다. 영어 중심으로 훈련된 모델의 한계다.

초보 친화도도 낮은 편이다. 허깅페이스에서 받아서 바로 쓰려면 파이썬 환경과 GPU 설정이 필요하다. 노코드 인터페이스는 없다.
