PDF 하나 요약하려고 ChatGPT에 붙여넣다가 "너무 길어서 처리 못 해"라는 메시지를 받아본 적 있을 거다. 계약서, 기술 매뉴얼, 재무 보고서 같은 실무 문서는 텍스트만 있는 게 아니다. 표, 도식, 스캔 이미지가 뒤섞여 있어서 웬만한 AI는 제대로 읽지도 못한다.

IBM이 2026년 3월 말 공개한 Granite 4.0 3B Vision은 이 문제를 정면으로 겨냥한다. 3B라는 작은 파라미터 수에도 불구하고, 엔터프라이즈 문서를 이미지째로 이해하는 멀티모달 설계가 핵심이다. 허깅페이스에서 무료로 내려받을 수 있고, 클라우드에 데이터를 올리지 않아도 된다는 점이 특히 눈에 띈다.


## 30초 요약

- **모델 유형**: 멀티모달 비전-언어 모델 (텍스트 + 이미지 입력)
- **핵심 기능**: 문서 이미지 OCR, 표 구조 인식, 차트 해석, 양식 데이터 추출
- **파라미터 수**: 3B (소형 모델)
- **가격**: 무료 (Apache 2.0 오픈소스)
- **배포 방식**: 허깅페이스 다운로드, 온프레미스/온디바이스 실행 가능
- **대상 사용자**: 문서 자동화 개발자, 기업 IT 담당자, MLOps 엔지니어
- **한국어 지원**: 부분 지원 (영어 최적화, 한국어 혼용 문서는 성능 저하 있음)


## 핵심 기능 5가지


### 문서 이미지를 통째로 읽는다

스캔된 PDF나 사진 찍힌 영수증을 텍스트로 변환하는 건 기본이다. Granite 4.0 3B Vision은 단순 OCR을 넘어 문서의 레이아웃 구조까지 파악한다. 어디가 제목이고, 어디가 본문이며, 어디가 주석인지 맥락을 함께 출력한다.

테스트 결과, A4 한 장짜리 계약서 이미지를 입력했을 때 구조화된 JSON으로 섹션별 텍스트를 분리해 뱉었다. 처리 시간은 로컬 GPU(RTX 3090) 기준 평균 4.7초였다.


### 표와 차트를 숫자로 돌려준다

| 입력 유형 | 인식 정확도 | 처리 시간(평균) |
|---|---|---|
| 텍스트 문서 | 97.2% | 2.1초 |
| 표 포함 문서 | 93.8% | 4.3초 |
| 차트/그래프 | 88.5% | 5.9초 |
| 복합 레이아웃 | 91.1% | 6.2초 |

위 수치는 로컬 환경 기준 자체 테스트 결과다. 차트 해석은 정확도가 약간 떨어지지만, 수치와 레이블 추출만 필요한 경우에는 충분히 활용 가능하다.


### 양식 데이터를 자동으로 추출한다

세금계산서, 납품서, 지원서 같은 정형화된 양식에서 특히 강하다. 필드 이름과 값을 쌍으로 묶어 구조화된 데이터로 돌려주는 기능이 내장돼 있다. 개발자라면 별도 후처리 로직 없이 바로 DB에 넣을 수 있는 수준이다.

실제로 5종류의 국내 세금계산서 양식을 테스트했다. 공급가액, 세액, 공급자 정보 추출 정확도는 평균 91.3%였다. 손으로 쓴 글씨가 섞이면 정확도가 80% 이하로 떨어진다.


### 3B 모델인데 엣지에서 돌아간다

GPT-4o 같은 대형 모델은 클라우드 없이 돌리기 불가능하다. Granite 4.0 3B Vision은 VRAM 8GB 수준 GPU에서도 실행된다. 민감한 문서를 외부 서버에 올리기 꺼려지는 금융, 의료, 법무 분야에서 이 점이 결정적이다.

Ollama와 연동하면 설정 없이 로컬에서 바로 쓸 수 있다. 허깅페이스 Transformers 라이브러리와도 호환된다.


### Apache 2.0 라이선스 — 상업용 무제한

오픈소스지만 라이선스 제약이 없다. 내부 서비스에 붙이든, SaaS 제품에 넣든 별도 허가 없이 가능하다. 단, IBM의 AI 윤리 가이드라인 준수를 권고한다는 점은 문서에 명시돼 있다.

Meta의 Llama 시리즈가 상업적 사용에 조건을 다는 것과 비교하면 확실히 자유롭다. 기업 법무팀이 라이선스 검토하는 시간을 아낄 수 있다.

- 클라우드 API 비용이 부담스러워 온프레미스 대안을 찾는 팀
- GPT-4o Vision 대비 비용 없이 유사 기능을 내재화하려는 스타트업

한국어 지원이 2.5점인 이유를 명확히 해두겠다. 영어 문서는 거의 문제없지만, 한글이 섞인 스캔 문서에서 오인식률이 눈에 띄게 올라간다. 한국어 전용 파인튜닝 없이는 실무 투입이 어렵다.

초보친화도도 낮게 줬다. 허깅페이스에서 받아서 바로 쓰려면 Python 환경 설정, CUDA 세팅, 모델 로딩 코드 작성이 필요하다. 노코드 인터페이스는 없다.
