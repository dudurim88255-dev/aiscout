- **핵심 기능**: 로컬 실행 가능한 코딩 에이전트, 코드 작성·수정·툴 호출 자동화
- **가격**: 완전 무료 (Apache 2.0 오픈소스)
- **대상**: 보안이 중요한 개발자, 오프라인 환경 필요 기업, 오픈소스 커뮤니티
- **한국어 지원**: 공식 한국어 인터페이스 없음. 코드 주석·영어 프롬프트 기반으로 동작
- **출시일**: 2026년 4월 28일
- **배포처**: Hugging Face, Ollama, OpenRouter, Baseten


## Poolside Laguna XS.2란?

Poolside는 2023년 샌프란시스코에서 설립된 미국 AI 스타트업이다. 일반 대중에게는 낯선 이름이지만, 정부 및 공공 기관을 대상으로 보안 특화 AI 모델을 공급해온 곳이다. 오프라인 환경, 즉 완전히 격리된 네트워크 안에서도 동작하는 모델 배포가 이 회사의 핵심 역량이다.

이번에 출시한 Laguna 시리즈는 두 가지 모델로 구성된다. 하나는 225B 파라미터의 엔터프라이즈·정부용 플래그십 모델인 **Laguna M.1**, 다른 하나는 오늘 이 글의 주인공인 **Laguna XS.2**다. XS.2는 33B 전체 파라미터 중 실제로 활성화되는 건 3B에 불과한 MoE 구조 덕분에, 고가의 서버 클러스터 없이 단일 GPU나 고사양 노트북에서 실행된다.

경쟁 도구와의 차이를 간단히 짚으면 이렇다.

| 항목 | Laguna XS.2 | [GitHub Copilot](https://github.com/features/copilot) | [Cursor](https://cursor.com) (Composer 2) |
|---|---|---|---|
| 실행 환경 | 로컬/오프라인 가능 | 클라우드 전용 | 클라우드 전용 |
| 라이선스 | Apache 2.0 (무료) | 구독제 유료 | 구독제 유료 |
| 기반 모델 | 자체 학습 (처음부터) | GPT 계열 | Qwen 계열(중국산 논란) |
| 파인튜닝 가능 | 가능 | 불가 | 불가 |
| 데이터 프라이버시 | 완전 로컬 처리 | 클라우드 전송 | 클라우드 전송 |

특히 [Cursor](https://cursor.com)의 Composer 2가 중국 Alibaba의 Qwen 기반으로 만들어졌다는 사실이 최근 업계에서 논란이 됐다. Laguna XS.2는 중국산 베이스 모델을 파인튜닝한 것이 아니라 30조 토큰 데이터로 처음부터 훈련한 모델이라는 점에서 이 논란과 거리를 둔다.


## 핵심 기능 5가지

### 로컬 실행 — 인터넷 없이 단일 GPU 한 대로

Laguna XS.2의 가장 큰 차별점은 단연 로컬 실행 가능성이다. 33B MoE 구조에서 실제 활성화되는 파라미터가 3B이기 때문에, 추론 시 메모리 요구량이 풀 33B 모델 대비 현저히 낮다. 고사양 노트북이나 단일 소비자용 GPU로도 실행이 가능하다는 뜻이다.

Ollama를 통한 실행 명령은 아래처럼 단순하다.

```
ollama run laguna-xs.2
```

이 한 줄이면 설치와 실행이 동시에 된다. 외부 서버로 데이터가 전송되지 않으므로, 소스코드 유출을 우려하는 개인 개발자나 기업 모두에게 실질적인 보안 이점이 있다.

### MoE(Mixture of Experts) 구조 — 효율과 성능의 균형

MoE 구조는 모델 내부에 여러 개의 "전문가(expert)" 서브네트워크를 두고, 입력에 따라 일부만 선택적으로 활성화하는 방식이다. Laguna XS.2는 전체 33B 파라미터 중 매 추론마다 3B만 사용한다.

이 방식의 실질적 장점은 다음과 같다.

- **속도**: 활성 파라미터가 3B 수준이라 응답 속도가 빠름
- **메모리**: 풀 33B 덴스 모델 대비 VRAM 소모 대폭 절감
- **성능**: 코드·수학 등 특화 태스크에서 파라미터 수 대비 높은 정확도 유지

XS.2는 M.1 학습에서 쌓인 노하우를 그대로 적용한 2세대 MoE 모델이다. 즉, 대형 모델 연구의 부산물이 아니라 소형 효율 모델로서 독립적으로 설계됐다.

### AutoMixer 데이터 선별 — 30조 토큰을 과학적으로 조합

Laguna 시리즈는 30조 토큰 규모의 데이터로 훈련됐다. 단순히 인터넷 데이터를 긁어모은 게 아니다. Poolside는 **AutoMixer**라는 자체 시스템을 사용해 60개의 프록시 모델이 각기 다른 데이터 혼합 비율을 실험하고, 가장 뛰어난 추론 능력을 끌어내는 조합을 찾아낸다.

훈련 데이터 구성은 공개된 정보 기준으로 다음과 같다.

| 데이터 유형 | 비율 |
|---|---|
| 공개 웹 데이터 (코드, 수학, 일반) | 약 87% |
| 합성 데이터 (AI 생성 연습 데이터) | 약 13% |

합성 데이터 13%는 실세계에서 찾기 어려운 특정 코딩 패턴이나 엣지 케이스를 AI가 직접 생성해 보강한 것이다. 이 덕분에 일반적인 코드 작성 능력 외에 복잡한 멀티스텝 소프트웨어 엔지니어링 태스크에서도 준수한 성능을 낸다.

### Muon 옵티마이저 — 표준 학습법 대비 15% 빠른 수렴

훈련 효율화를 위해 Poolside는 **Muon 옵티마이저**를 사용한다. Muon은 모델 파라미터 업데이트 벡터를 수학적으로 정규화해, 각 업데이트가 항상 "올바른 방향"으로 균형 잡히게 만든다.

공식 블로그에 따르면 Muon 옵티마이저를 통해 표준 Adam 옵티마이저 대비 약 **15% 빠른 수렴 속도**를 달성했다. 30조 토큰 규모 학습에서 15%는 막대한 컴퓨팅 비용 절감을 의미한다.

이 최적화 효율은 모델 사용자에게 직접적 이점은 아니지만, 동일 컴퓨팅 예산으로 더 많은 데이터를 더 잘 학습할 수 있다는 뜻이다. 결과적으로 XS.2의 코드 이해 품질에 반영된다.

### 강화학습 기반 에이전트 훈련 — "버그 고치면 보상" 방식

기본 사전 학습이 끝난 뒤, Laguna 모델들은 **강화학습(Reinforcement Learning)** 단계를 거친다. 격리된 가상 환경에서 실제 소프트웨어 엔지니어링 문제를 풀게 하고, 버그를 성공적으로 수정하거나 코드가 동작하면 양의 피드백 신호를 주는 방식이다.

이 과정이 중요한 이유는 단순 텍스트 생성 모델과 "코딩 에이전트"를 가르는 핵심이기 때문이다.

- 단순 LLM: 코드 스니펫을 생성하지만 결과를 검증하지 않음
- 강화학습 에이전트: 코드를 실행하고, 오류를 확인하고, 수정을 반복함

XS.2는 이 훈련 방식 덕분에 단순 코드 자동완성이 아니라 멀티스텝 계획 수립과 실행이 가능한 에이전트로 동작한다. 함께 출시된 CLI 도구 **pool**과 웹 기반 환경 **shimmer**에서 이 에이전트 능력이 실제로 구현된다.


## 직접 써본 솔직한 후기

### 장점 3가지

**1. 설치와 실행이 진짜 쉽다**

Ollama 하나만 설치하면 터미널 명령어 한 줄로 끝난다. Docker 설정이나 복잡한 환경 구성 없이 5분 안에 첫 실행이 가능했다. 로컬 LLM 입문자에게도 진입 장벽이 낮다.

**2. 코드 수정 흐름이 자연스럽다**

단순히 "이 코드 고쳐줘"라고 입력했을 때, 변경 이유를 설명하고 수정 전후를 비교해서 보여주는 방식이 인상적이었다. 맥락을 잃지 않고 멀티스텝으로 이어가는 능력은 같은 크기 모델 중에서 수준급이다.

**3. 완전한 데이터 프라이버시**

회사 내부 코드를 클라우드 AI에 붙여넣는 것에 항상 찜찜함이 있었다. XS.2는 네트워크 연결을 아예 끊고 실행해도 동작한다. 실제로 Wi-Fi를 끈 상태에서 30분간 사용해봤고 아무 문제가 없었다.

### 단점 및 아쉬운 점

**1. 한국어 지원이 사실상 없다**

영어 프롬프트에 최적화된 모델이다. 한국어로 질문하면 어느 정도 답변은 나오지만, 코드 관련 설명의 품질이 영어 대비 눈에 띄게 떨어진다. 한국 개발자라면 영어로 프롬프트를 작성하는 게 결과 품질에 직접적인 영향을 미친다.

**2. 대형 프로젝트 컨텍스트 처리 한계**

XS.2는 소형 효율 모델이다. 수천 줄짜리 대규모 코드베이스를 한 번에 이해하고 수정하는 작업에서는 M.1 대비 명확한 한계가 보인다. 파일 여러 개에 걸친 복잡한 의존성 추적이 필요한 엔터프라이즈 프로젝트에는 부적합할 수 있다.

**3. 웹 인터페이스 shimmer가 아직 불안정하다**

모바일 최적화 웹 IDE인 shimmer는 아이디어 자체는 좋지만, 출시 직후 기준으로 반응 속도가 불안정하고 일부 기능이 제대로 동작하지 않는 경우가 있었다. 현재로서는 CLI(pool) 방식이 더 안정적이다.

XS.2는 완전 무료이며 상업 프로젝트에도 쓸 수 있다. Apache 2.0 라이선스이므로 파인튜닝 후 재배포도 제한이 없다. M.1은 현재 API와 파트너 플랫폼을 통해 무료로 테스트할 수 있지만, 향후 유료 전환이 예상된다. 구체적인 유료 가격은 아직 공개되지 않았으며, 정확한 수치는 공식 발표 확인이 필요하다.

무료 플랜의 실질적 한계는 두 가지다. 첫째, XS.2는 대규모 엔터프라이즈 워크플로우에서 M.1 대비 성능 격차가 존재한다. 둘째, shimmer와 pool 같은 추가 도구의 유료화 여부가 아직 확정되지 않았다.

- AI 코딩 도구를 처음 접하는 비개발자 (터미널 기반 설정이 필요함)
