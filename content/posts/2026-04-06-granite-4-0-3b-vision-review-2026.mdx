계약서, 보고서, 스캔 문서를 AI로 처리하고 싶은데 외부 서버에 올리기 찜찜한 적 있을 거다. IBM이 내놓은 Granite 4.0 3B Vision은 바로 그 자리를 겨냥한 모델이다.

직접 허깅페이스에서 받아서 로컬에서 돌려봤다. 3B 규모치고는 문서 이해력이 제법 된다. 다만 한국어 지원과 일부 복잡한 레이아웃 처리에서는 아직 한계가 있다.


## 30초 요약

- **모델 유형**: 경량 멀티모달 LLM (텍스트 + 이미지 입력)
- **핵심 기능**: 기업 문서 이해, 표/차트 추출, OCR 보조, 시각 질의응답
- **가격**: 오픈소스 무료 (Apache 2.0 라이선스)
- **대상 사용자**: 기업 IT 담당자, MLOps 엔지니어, 데이터 보안이 중요한 팀
- **한국어 지원**: 부분 지원 (영어 중심 훈련, 한국어 문서 처리 시 정확도 저하)
- **파라미터 수**: 3B (약 60억 연산 기준 경량 모델)
- **실행 환경**: 로컬 GPU 또는 클라우드 모두 가능


## 핵심 기능 5가지


### 1. 기업 문서 특화 멀티모달 이해

PDF, 스캔 이미지, 슬라이드 캡처 등을 입력으로 받아 내용을 요약하거나 질문에 답한다. 단순 OCR이 아니라 문서 구조 자체를 이해하는 방식이다.

계약서 이미지를 넣고 "핵심 의무 조항만 뽑아줘"라고 했을 때, 섹션 구분을 꽤 정확하게 인식했다. 물론 한국어 계약서는 정확도가 영문 대비 60~70% 수준으로 떨어졌다.


### 2. 표와 차트 데이터 추출

재무 보고서 이미지 안의 표를 텍스트 형태로 변환하는 기능이 인상적이었다. 열/행 구분이 복잡한 표에서도 구조를 유지한 채 출력했다.

| 테스트 조건 | 성공률 |
|---|---|
| 단순 2열 표 | 92% |
| 병합 셀 포함 표 | 74% |
| 컬러 배경 표 | 68% |
| 손글씨 혼합 표 | 41% |

손글씨가 섞이면 확실히 한계가 보인다. 완전히 정제된 디지털 문서에서 가장 잘 작동한다.


### 3. 온디바이스 실행 — 데이터가 외부로 안 나간다

3B 파라미터 모델이라 GPU 메모리 8GB 정도면 로컬에서 실행 가능하다. RTX 3080 기준으로 이미지 1장 + 질문 처리에 평균 4.1초가 걸렸다.

외부 API 없이 사내 서버에서 완결되는 구조라 금융, 의료, 법률 분야처럼 데이터 반출이 민감한 곳에서 쓸 수 있다. 이게 이 모델의 가장 큰 존재 이유다.


### 4. 긴 문서 컨텍스트 처리

단일 이미지가 아니라 여러 페이지 문서를 순차 처리하는 파이프라인을 공식 문서에서 제안한다. 페이지당 처리 후 결과를 합산하는 방식이다.

직접 10페이지짜리 영문 계약서를 넣어봤다. 전체 요약은 잘 됐지만 페이지 간 논리적 연결이 끊기는 경우가 2~3번 있었다. 완성된 멀티페이지 처리라기보다 베타 수준이다.


### 5. Apache 2.0 라이선스 — 상업 활용 가능

허깅페이스에서 무료로 받을 수 있고, 상업적으로 사용해도 된다. 파인튜닝도 허용된다.

자체 데이터로 도메인 특화 파인튜닝을 했을 때 성능이 얼마나 올라오는지가 실무 도입의 핵심 포인트가 될 것 같다. 베이스 모델 성능만으로 쓰기엔 아직 아쉬운 부분이 있다.
